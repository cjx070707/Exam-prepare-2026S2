# COMP5329 复习前必看

> 每次开始复习前读这个文件。它解释了整套复习体系的逻辑、文件结构和使用顺序。

---

## 一、考试题型

| 题型 | 要求 |
|------|------|
| 单选题 | 关键事实、概念区分、历史节点 |
| 简答题 | 2–5 句话，术语准确 |
| 手写代码题 | 从零手写关键算法（无 autograd） |
| 论述题 | 结构化比较分析（A 在条件 C 下优于 B，为什么）|

## 二、三文件体系

复习资料分三类文件，各有分工：

### 📘 复习笔记（theory，考点理论）
```
复习笔记/
├── 第一组 基础知识.md      ← W2（神经网络）+ W3（正则化）
├── 第二组 结构化数据.md     ← W4（CNN）+ W5（GNN）
├── 第三组 序列数据.md       ← W7（RNN/LSTM）+ W8（Transformer/BERT/ViT/SSM）
└── 第四组 超越监督学习.md   ← W9（多模态）+ W10（RL）+ W11（SSL）+ W12（生成模型）
```
- **只含理论、公式、推导、概念对比**
- 不含代码（代码在速查表）

### 💻 手写代码速查表.md（coding，考前默写）
```
手写代码速查表.md
```
- 来源：Tutorial 主 notebook 的 coding exercises
- 含：init、激活函数 backward、HiddenLayer backward、Loss backward、Adam、BN、Dropout、Conv、Pool、GCN
- 用法：考前逐块默写，检查 shape 和公式

### 📝 考点练习题.md（exam questions，做题）
```
（待创建）考点练习题.md
```
- 来源：Tutorial Self_Study notebook 末尾的 "Exam-style Questions"（Week7–12 有）
- 每周 3 道跨组综合题，含答案 sketch
- Group 1+2 范围（W2–W5）暂无 Self_Study 题，用 `/study-quiz` 命令生成

## 三、材料来源规则（严格遵守）

| 材料 | 用途 | 是否用作来源 |
|------|------|-------------|
| Lecture PDF（每周一个）| 复习笔记的唯一内容来源 | ✅ 必须 |
| Tutorial 主 notebook | 手写代码速查表的来源 | ✅ 用于代码 |
| Self_Study notebook | 考点练习题的来源 | ✅ 仅题目部分 |
| Notes PDF（W2-Notes.pdf 等）| 课外阅读，不属于 lecture | ❌ 不用 |
| 训练知识补充 | — | ❌ 绝对禁止 |

**关键原则：每条笔记内容必须能追溯到实际读过的 lecture PDF 对应页面。**

## 四、四组复习顺序

```
第一组 → 第二组 → 第三组 → 第四组
（W2+W3）  （W4+W5）  （W7+W8）  （W9–W12）
```

每组之间有问题链（上一组留下的问题驱动下一组的解决方案）：
- 线性模型不够 → MLP + 反向传播 → 过拟合 → 正则化（第一组）
- MLP 忽略空间结构 → CNN → 需要图结构 → GCN（第二组）
- 静态模型无法处理序列 → RNN → 梯度消失 → LSTM → Transformer → SSM（第三组）
- 监督学习需要标注 → 多模态/RL/SSL/生成模型（第四组）

---

## 五、每次复习 Session 的流程（4 步）

```
Step 1  回忆问题链（不看笔记，口述本组的"问题→方案→新问题"逻辑）
Step 2  公式默写（不看笔记，写出本组所有 ★★★ 公式）
Step 3  代码默写（打开速查表对应部分，逐块默写，检查 shape）
Step 4  做题（打开练习题文件，先做 ★★★ 相关题，再做跨组综合题）
```
