## 这一组在讲什么

前三组的所有方法都有一个共同前提：**需要标注好的标签**。但标签是昂贵、稀缺、噪声大的。

这一组讲的是四条"不依赖标签（或减少对标签依赖）"的路：

| 方向 | 监督信号来自哪里 | 代表方法 |
|------|---------------|---------|
| **多模态** | 不同模态之间互相监督 | CLIP, BLIP-2, LLaVA |
| **强化学习** | 环境的奖励信号 | Q-Learning, PPO, RLHF |
| **自监督学习** | 数据本身的结构 | MAE, SimCLR, BYOL, DINO |
| **生成模型** | 学习数据分布 | GAN, VAE, DDPM, Flow Matching |

> [!info] 问题链
> **标签稀缺/昂贵** → 四条出路：(1) **多模态预训练**（不同模态互监督，CLIP / MLLM）→ (2) **环境奖励替代标签**（RL → RLHF 对齐 LLM）→ (3) **数据本身作监督信号**（SSL：对比学习 / MAE / CLIP）→ (4) **学习数据分布直接生成**（GAN → 梯度消失 → WGAN / VAE / Diffusion / Flow Matching）

## Week 9 — 多模态大模型（MLLM）

### 三组件架构 `★★`

> [!tip] LLaVA 用 MLP projection 作为 Connector，BLIP-2 用 Q-Former——两者的核心区别是什么？Q-Former 的 learnable query 解决了什么问题？训练时通常冻结哪些组件、只训练哪些？

几乎所有 MLLM 都遵循这个架构：

```
图像输入
   ↓
[Vision Encoder]        ← 把图片变成特征向量序列（常用 CLIP-ViT）
   ↓
[Connector Module]      ← 把视觉特征"翻译"成语言模型能理解的 token
   ↓
[Language Model]        ← Transformer Decoder，混合处理视觉+文本 token
   ↓
文字输出（或图像 token）
```

**Vision Encoder**：用 CLIP 预训练的 ViT，已经学会了视觉-语言对齐的表示。

**Connector 的两大类设计**：

**Token-level fusion**（把视觉特征变成 token 拼到文本序列前）：
- **MLP/Projection**（LLaVA）：每个 patch feature 直接线性投影，最简单
- **Q-Former**（BLIP-2）：32 个可学习 query vector 通过 cross-attention 提炼视觉特征，压缩 token 数量

**Feature-level fusion**（在 LLM 内部注入视觉信息）：
- **Cross-attention insertion**（Flamingo）：在 LLM 每层插入 cross-attention，视觉信息在各深度影响语言理解
- **Layer-wise adapters**（CogVLM、LLaMA-Adapter）：在 LLM 各层插入可训练的视觉适配模块

**Language Model**：冻结的（或 LoRA 微调的）大型 decoder-only Transformer。

> **注**：Vision Mamba（Vim）是将 Mamba 用于视觉的变体——与 ViT 一样切 patch，但用双向 SSM（Forward SSM + Backward SSM）替代 self-attention，保留 O(N) 复杂度的同时获得双向上下文。MLLM 中的 Vision Encoder 也可用 Vim 替代 ViT。`★`

<span style="color: #9b59b6">💡 **白话理解**：三个组件解决三个独立问题：Vision Encoder 把图片变成向量，Connector 把视觉语言"翻译"成 LLM 能懂的格式，LLM 做最终理解。Connector 是设计差异最大的地方：MLP 简单但 token 数多，Q-Former 用 learnable query 先做一次"摘要"再传给 LLM。</span>

<span style="color: #e67e22">*三组件架构解决了"怎么连接视觉和语言"，但训练整个 LLM 代价太高——LoRA 是让微调变得实际可行的关键技术。*</span>

### LoRA：高效微调 `★★★`

> [!tip] 你需要把一个 7B 参数的语言模型微调到医疗问答任务，但 GPU 显存只够放原始模型的参数，没有空间存梯度——LoRA 如何在几乎不增加显存的情况下完成微调？为什么 B 矩阵必须初始化为全零？

把大模型全参数微调代价太高，LoRA 的核心洞察：**微调时权重的更新是低秩的**。

原始权重 $W_0 \in \mathbb{R}^{d \times k}$ 冻结，只训练低秩分解：

$$W = W_0 + \Delta W = W_0 + BA, \quad B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times k}, \quad r \ll \min(d, k)$$

参数量从 $dk$ 降到 $r(d+k)$，通常节省 100-500 倍。

**初始化**：$B=0$，$A$ 随机初始化 → 初始时 $\Delta W = BA = 0$，从预训练模型的精确状态开始微调。

<span style="color: #9b59b6">💡 **白话理解**：LoRA 的核心假设是"微调时需要改变的方向是低维的"——不需要动所有参数，只需要在原始权重上叠加一个"低秩补丁"（BA）。B=0 初始化保证训练开始时补丁为零，从预训练的精确状态出发，而不是从随机噪声开始。</span>

### KV Cache：推理加速 `★`

自回归生成时，每生成一个新 token，需要重新计算整个序列的 K 和 V。

KV Cache：把已计算的所有位置的 K、V 存起来，下一步只计算新 token 的 K、V，然后拼接到缓存里：

$$\text{output}_t = \sum_{j=1}^{t} \alpha_{t,j} V_j, \quad \text{其中 } K_j, V_j\ (j < t) \text{ 从缓存读取}$$

**代价**：内存正比于 $O(N \cdot L \cdot d)$，长序列时内存压力大。

### MoE：用专家路由扩大容量 `★`

Mixture of Experts：维护 $E$ 个独立的 FFN "专家"，每个 token 只激活 top-$k$ 个专家：

$$\text{MoE}(x) = \sum_{i \in \text{top-}k} g_i(x) \cdot f_i(x)$$

**好处**：总参数量 $\propto E$，但每 token 计算量 $\propto k$（$k \ll E$）。大容量，低推理成本。

> **考试提示**：LoRA、KV Cache、MoE 是"效率三件套"，选择题和短答题都可能考它们各自解决什么问题。

<span style="color: #e67e22">*多模态和大模型解决的是"如何利用现有知识"。但有些任务没有固定的正确答案——下棋、控制机器人、对齐 AI——只有"这步走得好不好"的反馈。强化学习用环境奖励替代标签。*</span>

## Week 10 — 强化学习（RL）

### MDP：RL 的数学框架 `★★`

> [!tip] 一个自动驾驶系统需要在十字路口做决策——用 MDP 的五元组描述这个场景中的 $S$、$A$、$r$、$\gamma$ 各是什么？$\gamma$ 设为 0.99 和 0.5 分别意味着什么驾驶风格？

**马尔可夫决策过程**（Markov Decision Process）：$(S, A, P, r, \gamma)$

| 符号 | 含义 |
|------|------|
| $S$ | 状态空间 |
| $A$ | 动作空间 |
| $P(s' \| s, a)$ | 状态转移概率 |
| $r(s, a)$ | 即时奖励 |
| $\gamma \in [0,1)$ | 折扣因子（$\gamma$ 越大越有远见）|

**策略** $\pi(a \| s)$：给定状态，输出动作的概率分布。

**目标**：找到最优策略 $\pi^*$，使期望折扣回报最大：

$$J(\pi) = \mathbb{E}_\pi\!\left[\sum_{t=0}^{\infty} \gamma^t r_t\right]$$

### 价值函数

**状态价值函数**：从状态 $s$ 出发、遵循策略 $\pi$ 的期望回报：

$$V^\pi(s) = \mathbb{E}_\pi[G_t \mid s_t = s]$$

**动作价值函数**：在状态 $s$ 执行动作 $a$ 后、遵循策略 $\pi$ 的期望回报：

$$Q^\pi(s, a) = \mathbb{E}_\pi[G_t \mid s_t = s, a_t = a]$$

**优势函数**：动作 $a$ 比平均水平好多少：

$$A^\pi(s, a) = Q^\pi(s, a) - V^\pi(s)$$

**Bellman 方程**（递推结构）：

$$V^\pi(s) = \mathbb{E}_{a \sim \pi}\!\left[r(s,a) + \gamma V^\pi(s')\right]$$

$$Q^*(s,a) = \mathbb{E}\!\left[r + \gamma \max_{a'} Q^*(s', a')\right]$$

<span style="color: #9b59b6">💡 **白话理解**：$V$ 是"从这个状态出发，遵循当前策略平均能拿多少分"；$Q$ 是"在这个状态做这个动作，再遵循当前策略平均能拿多少分"；$A = Q - V$ 是"这个动作比平均水平好多少"——正数就增加概率，负数就减少。</span>

### Q-Learning `★★`

直接学最优动作价值函数 $Q^*$：

$$Q(s,a) \leftarrow Q(s,a) + \alpha \underbrace{\left[r + \gamma \max_{a'} Q(s', a') - Q(s,a)\right]}_{\text{TD Error}}$$

- **Off-policy**：用 $\max_{a'} Q(s', a')$ 而不管实际采用什么策略
- **Bootstrapping**：用自己当前的估计值更新自己

**探索 vs 利用（Exploration vs Exploitation）**：

- **Random policy**：纯探索，随机选动作
- **Greedy policy**：纯利用，$\pi(s) = \arg\max_a Q(s,a)$
- **ε-greedy**：以概率 $\varepsilon$ 随机选动作（探索），以概率 $1-\varepsilon$ 选最优动作（利用）

**DQN**（Deep Q-Network）：状态空间过大时（如 Atari 游戏：$256^{210 \times 160}$ 个状态），用神经网络近似 $Q(s,a) = f(s,a,w)$。

- **输入**：最近 4 帧堆叠（4×84×84），**输出**：所有动作的 Q 值（不是单个 Q 值）
- **DQN Loss**（MSE on Bellman target）：
  $$l = \left(r + \gamma \max_{a'} Q_w(s', a') - Q_w(s,a)\right)^2$$
- **Divergence issues**（不加 trick 会发散的两个原因）：
  - **Correlations between samples**：连续 transition 相互关联，违反 i.i.d. 假设
  - **Non-stationary targets**：target 用的是同一个网络，边学边变，追移动靶
- **Experience Replay**：把 $(s_t, a_t, r_{t+1}, s_{t+1})$ 存入 replay buffer，uniformly 随机采样 → 打破时序相关性
- **Target Network**：用一个慢更新的副本计算 target，固定 N 步后才同步 → 稳定训练目标

<span style="color: #9b59b6">💡 **白话理解**：Q-Learning 就是不断更新"做这个动作能拿多少分"的估计——实际拿到的分数和预期不同（TD Error），就往真实值方向修正。DQN 的两个 trick 各解决一个训练不稳定问题：Replay 打破"连续经验互相干扰"，Target Network 防止"自己追自己不收敛"。</span>

### Policy Gradient `★★`

不学价值，直接优化策略参数 $\theta$。**策略梯度定理**：

$$\nabla_\theta J(\theta) = \mathbb{E}\!\left[\nabla_\theta \log \pi_\theta(a_t \mid s_t) \cdot Q^\pi(s_t, a_t)\right]$$

实践中用优势函数 $A^\pi$ 替代 $Q^\pi$（方差更低）：

$$\nabla_\theta J(\theta) = \mathbb{E}\!\left[\nabla_\theta \log \pi_\theta(a \mid s) \cdot A^\pi(s, a)\right]$$

**直觉**：如果这个动作比平均好（$A > 0$），就增大它的概率；如果比平均差（$A < 0$），就减小概率。

### PPO（Proximal Policy Optimization） `★★`

策略梯度的问题：步长太大会让策略崩溃。PPO 用 clip 限制每步的策略变化幅度：

$$L^\text{CLIP}(\theta) = \mathbb{E}\!\left[\min\!\left(r_t(\theta) A_t,\ \text{clip}(r_t(\theta), 1-\varepsilon, 1+\varepsilon) A_t\right)\right]$$

其中 $r_t(\theta) = \frac{\pi_\theta(a|s)}{\pi_{\theta_\text{old}}(a|s)}$ 是新旧策略的比值。

<span style="color: #9b59b6">💡 **白话理解**：策略梯度的直觉是"好的动作增概率，坏的动作减概率"。PPO 是加了一个安全阀——新旧策略不能差太多（比值限制在 $[1-\varepsilon, 1+\varepsilon]$），防止一步走太大导致策略崩溃。</span>

### RLHF（Reinforcement Learning from Human Feedback） `★★★`

> [!tip] LLM 生成文字时，State/Action/Reward 分别对应什么？PPO 在 RLHF 里需要同时维护几个模型，各自的作用是什么？为什么要加 KL 惩罚项？

**语言生成作为 RL 问题**（p25）：

| RL 概念 | LLM 对应 |
|--------|---------|
| State $s_t$ | prompt + 已生成 tokens $(x, y_1, \ldots, y_{t-1})$ |
| Action $a_t$ | 下一个 token $y_t$ |
| Policy $\pi_\theta$ | LM 的 next-token 分布 |
| Episode | 一次完整生成（prompt → EOS）|
| Reward | 人类偏好分数 $r_\phi(x, y)$（仅在最后一步给出）|

**RLHF 三阶段 Pipeline**（InstructGPT/ChatGPT 标准范式）：

**Stage 1 — SFT**：在人工示范数据上有监督微调：

$$\mathcal{L}_\text{SFT}(\theta) = -\mathbb{E}_{(x,y) \sim \mathcal{D}_\text{demo}}\!\left[\sum_{t=1}^{T} \log \pi_\theta(y_t \mid x, y_{<t})\right]$$

产出 $\pi^\text{SFT}$，作为 RM 和 Stage 3 的初始化，以及 KL 惩罚的参考策略。

**Stage 2 — Reward Model**：收集人类偏好对 $(x, y_w, y_l)$（$y_w$ 优于 $y_l$），用 **Bradley-Terry 模型**建模：

$$P(y_w \succ y_l \mid x) = \sigma\!\left(r_\phi(x, y_w) - r_\phi(x, y_l)\right)$$

RM 损失（最大似然）：

$$\mathcal{L}_\text{RM}(\phi) = -\mathbb{E}_{(x,y_w,y_l)}\!\left[\log \sigma\!\left(r_\phi(x,y_w) - r_\phi(x,y_l)\right)\right]$$

架构：SFT 模型最后一层替换为线性投影→标量输出。

**Stage 3 — RL Fine-Tuning（RLHF Objective）**：

$$\max_\theta\, \mathbb{E}_{x,\, y \sim \pi_\theta}\!\left[\underbrace{r_\phi(x, y)}_{\text{偏好信号}} - \underbrace{\beta \cdot D_\text{KL}(\pi_\theta \| \pi_\text{ref})}_{\text{drift 惩罚}}\right]$$

**KL 惩罚的作用**：防止 reward hacking（策略跑偏去骗 RM），保持生成质量接近 SFT 模型。$\beta$ 控制 trade-off。

**PPO for LLMs — 需要 4 个模型同时在显存中**：

| 模型 | 角色 | 可训练？ |
|------|------|---------|
| Policy $\pi_\theta$ | 被优化的 LLM | ✅ |
| Reference $\pi_\text{ref}$ | 冻结的 SFT 副本，KL 锚点 | ❌ 冻结 |
| Reward $r_\phi$ | 对完整输出打分 | ❌ 冻结 |
| Value $V_\psi$ | Critic，用于估计 Advantage | ✅ |

> 7B 模型 × 4 → ~28B 参数，加优化器状态 **100+ GB GPU 显存**。这是更简单替代方案（DPO、GRPO）的主要动机。

**Advantage 计算**（p31）：

$$\hat{A}_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$

PPO clip 公式（同上节）：$\mathcal{L}_t^\text{CLIP} = \min(\rho_t \hat{A}_t,\ \text{clip}(\rho_t, 1-\varepsilon, 1+\varepsilon)\hat{A}_t)$

<span style="color: #9b59b6">💡 **白话理解**：RLHF 解决的是"人类偏好难以量化成标签"的问题——人只能说"A 比 B 好"，不能给出精确分数。Bradley-Terry 把"相对好"量化成标量奖励。KL 惩罚是"安全绳"——防止 LLM 为了骗高分而生成人类偏好模型没见过的奇怪内容（reward hacking）。</span>

### DPO（Direct Preference Optimization） `★★★`

> [!tip] DPO 跳过了 Reward Model，那偏好信号从哪里来？为什么说 DPO 是对 RLHF 目标的"等价优化"？

**动机**：PPO 需要 4 个模型，训练不稳定，超参多。DPO 发现：RLHF 的 KL 约束目标有**闭式最优解**，可以直接绕过 RM 训练。

**关键推导**（p36）：RLHF 目标的最优策略：

$$\pi^*(y|x) = \frac{1}{Z(x)}\pi_\text{ref}(y|x)\exp\!\left(\frac{r(x,y)}{\beta}\right)$$

反解出 reward（reparameterization）：

$$r(x, y) = \beta \log \frac{\pi^*(y|x)}{\pi_\text{ref}(y|x)} + \beta \log Z(x)$$

**两个回复相比，$Z(x)$ 约掉**，代入 Bradley-Terry，得到 **DPO Loss**：

$$\mathcal{L}_\text{DPO} = -\mathbb{E}_{(x,y_w,y_l)}\!\left[\log \sigma\!\left(\beta \left(\log \frac{\pi_\theta(y_w|x)}{\pi_\text{ref}(y_w|x)} - \log \frac{\pi_\theta(y_l|x)}{\pi_\text{ref}(y_l|x)}\right)\right)\right]$$

- 第一项 = 优胜回复的隐式奖励（↑ 增大概率）
- 第二项 = 落败回复的隐式奖励（↓ 减小概率）
- 本质是**偏好对上的二元交叉熵**，监督式训练，稳定。

**DPO vs RLHF 对比**：

| 方面 | RLHF (PPO) | DPO |
|------|-----------|-----|
| 流程 | SFT → RM → PPO | SFT → DPO fine-tuning |
| 显存 | 4 个模型 | 2 个模型 ($\pi_\theta, \pi_\text{ref}$) |
| 在线生成？ | 是（需要持续采样）| 否（离线固定偏好数据集）|
| 训练稳定性 | 低 | 高（类监督训练）|
| 代表系统 | ChatGPT | Llama 2/3 |

**DPO 局限**（p38）：离线分布 mismatch，对 $\beta$ 敏感，无显式 reward 可复用，对标注噪声敏感。

<span style="color: #9b59b6">💡 **白话理解**：DPO 的核心洞察是——RM 其实可以用 policy 本身来隐式表达。当两个回复 $Z(x)$ 相同时约掉，reward gap 就直接变成了 policy log-ratio gap。整个训练变成：让 $\pi_\theta$ 在 winner 上比 $\pi_\text{ref}$ 概率更高、在 loser 上更低——不需要单独的 RM。</span>

---

### SLiC-HF `★`

比 DPO 更简单的偏好优化，直接用排名 loss：

**Max-margin ranking loss**（hinge loss）：
$$\mathcal{L}_\text{rank} = \max(0,\ \delta - \log\pi_\theta(y_w|x) + \log\pi_\theta(y_l|x))$$

让 winner 的 log-likelihood 至少比 loser 高 $\delta$（margin）。不涉及 $\pi_\text{ref}$ 的 log-ratio，更简单但无理论等价性保证。

### GRPO（Group Relative Policy Optimization） `★★`

> [!tip] GRPO 是如何在不训练 Critic 的情况下估计 Advantage 的？DeepSeek-R1 为什么选择 GRPO 而不是 PPO？

**动机**：PPO 需要 Value Model（Critic），参数量和 Policy 相当 → **显存翻倍**。GRPO 用组内统计替代 Critic。

**核心思路**：对每个 prompt $x$，采样 $G$ 个输出 $\{o_1, \ldots, o_G\}$，用 RM 打分 $\{r_1, \ldots, r_G\}$，**组内标准化**估计 Advantage：

$$\hat{A}_i = \frac{r_i - \text{mean}(\{r_1, \ldots, r_G\})}{\text{std}(\{r_1, \ldots, r_G\})}$$

不需要任何额外网络，只用"这组回复中谁比平均好"作为奖励基准。

**GRPO Objective**（PPO-style clipping + KL）：

$$J_\text{GRPO}(\theta) = \frac{1}{G}\sum_{i=1}^{G}\frac{1}{|y_i|}\sum_{t=1}^{|y_i|}\left[\min\!\left(\rho_{i,t}\hat{A}_i,\ \text{clip}(\rho_{i,t}, 1-\varepsilon, 1+\varepsilon)\hat{A}_i\right) - \beta D_\text{KL}\right]$$

其中 $\rho_{i,t} = \pi_\theta(y_{i,t}|x, y_{i,<t}) / \pi_{\theta_\text{old}}(y_{i,t}|x, y_{i,<t})$ 是 per-token importance ratio。$\hat{A}_i$ 是 **outcome-level advantage**（整个回复共享同一个值）。

**GRPO = REINFORCE + group-mean baseline + PPO clipping**

**PPO vs GRPO 模型数量**：

| 组件 | PPO | GRPO |
|------|-----|------|
| Policy | ✅ | ✅ |
| Reference | ✅ | ✅ |
| Reward | ✅ | ✅ |
| Value/Critic | ✅ | ❌（用组统计替代）|
| **总计** | **4 个** | **3 个** |

**DeepSeek-R1**（2025）：在 671B MoE 模型上用 GRPO。R1-Zero：仅 GRPO + rule-based reward（无 SFT），出现了 emergent chain-of-thought reasoning！

<span style="color: #9b59b6">💡 **白话理解**：GRPO 的核心是"用同组其他答案当基准"——如果这次回复比组内平均好（$\hat{A}_i > 0$），就增大概率；比平均差就减小。免去了专门的 Critic 网络，节省了 25% 的显存。代价是每次要多生成 G=16–64 个样本，推理成本上升。</span>

## Week 11 — 自监督学习（SSL）

### 核心动机

有标注数据少，无标注数据多。能不能用无标注数据学到好的表示？

**自监督学习**：设计一个"借口任务"（pretext task），让数据自己监督自己。

### 生成式 SSL：重建输入 `★★`

**Autoencoder**：encoder 把输入压缩到瓶颈 $z$，decoder 重建原始输入：

$$\mathcal{L}_\text{AE} = \|x - \hat{x}\|^2$$

**问题**：decoder 太强时，只学像素级重建，encoder 不需要学到语义。

**MAE（Masked Autoencoder）**：遮掉 75% 的 patch，只用剩下 25% 的 patch 重建。任务更难 → encoder 必须学全局语义。

<span style="color: #9b59b6">💡 **白话理解**：MAE 就是"看图说话的极端版"——给你 25% 的拼图碎片，让你还原整张图。任务难到必须理解图片的全局语义，不能靠局部插值。75% 的 masking ratio 是精心设计的——太低任务太简单，学不到好表示。</span>

### 对比学习：通过比较来学习 `★★★`

> [!tip] 你的对比学习模型训练时 loss 降得很快，但下游任务效果一般——你怀疑是负样本质量问题。InfoNCE 的梯度显示"难负样本"权重更大是什么意思？温度参数 τ 设得太小会产生什么副作用？

**核心思想**：同一张图的两种增强（正样本对）应该表示相近，不同图的表示应该分开。

**InfoNCE Loss**：

$$\mathcal{L} = -\log \frac{\exp(q \cdot k^+ / \tau)}{\sum_j \exp(q \cdot k_j / \tau)}$$

其中 $q$ 是 query，$k^+$ 是正样本，$k_j$ 包括正负样本，$\tau$ 是温度参数。

**梯度分析**：$\frac{\partial \mathcal{L}}{\partial q} = -\frac{1}{\tau}k^+ + \frac{1}{\tau}\sum_j p_j k_j$

→ 正样本吸引，负样本排斥，且权重 $p_j$ 对**难负样本**（和 $q$ 相似的负样本）更大。

**SimCLR**：最简单的对比学习。同一图的两种随机增强作为正样本，batch 内其他图作为负样本。需要非常大的 batch size。

**MoCo**：用 momentum encoder 维护一个负样本队列，不依赖大 batch：

$$\theta_k \leftarrow m \theta_k + (1-m) \theta_q, \quad m \approx 0.999$$

<span style="color: #9b59b6">💡 **白话理解**：对比学习就是"近朱者赤，近墨者黑"——同一张图的两个视角（不同裁剪、颜色抖动等）应该靠近，不同图应该推开。难负样本（看起来像但其实不同的图）梯度权重更大，因为它们是最容易让模型"看走眼"的情况。</span>

### 非对比学习：不需要负样本 `★★`

**BYOL**：teacher-student 架构，student 预测 teacher 的表示：

$$\mathcal{L}_\text{BYOL} = \|q_\theta(x_1) - \text{sg}(z_\xi(x_2))\|^2$$

`sg` 是 stop-gradient。**不对称性**（stop-gradient + predictor）防止了 collapse，不需要负样本。

**SimSiam**：去掉了 momentum teacher（No EMA, No separate target），两个分支共享权重，更简单：

$$\mathcal{L} = -\frac{1}{2}\left[\text{sim}(p_1, \text{sg}(z_2)) + \text{sim}(p_2, \text{sg}(z_1))\right]$$

其中 $p$ = predictor 输出，$z$ = encoder 输出，$\text{sg}$ = stop-gradient，$\text{sim}$ = cosine similarity。

**Stop-gradient ablation**（PDF p43）：
- **Without sg**：loss 收敛到 −1 并平坦（两分支collapse到相同常数向量），output std → 0，kNN acc → 0
- **With sg**：loss 正常下降，output std 稳定在 $1/\sqrt{d}$，kNN acc 上升至 50+
- **关键**：即使有 sg，去掉 predictor 也会 collapse——**predictor 和 stop-gradient 都是必须的**

**BYOL vs SimSiam 比较**（PDF p51）：

| 方面 | BYOL | SimSiam |
|------|------|---------|
| Target | EMA copy | shared weights |
| EMA | yes (τ: 0.996→1.0) | none |
| Predictor | yes（essential）| yes（essential）|
| Stop-grad | on target | on target |
| Complexity | higher | minimal |
| IN-1K LP | 74.3% | 71.3% |

**Minimal recipe**：shared encoder + predictor + stop-gradient。EMA 有帮助但不是必须的。

**DINO**：用 cross-entropy 让 student 匹配 teacher 的 softmax 分布：

$$\mathcal{L}_\text{DINO} = -\sum_k p_t^{(k)} \log p_s^{(k)}$$

**架构**：student $g_{\theta_s}$ + teacher $g_{\theta_t}$（EMA），两者都输出 softmax 分布（probability simplex 上的 soft assignment over latent prototypes）。Loss = $-p_2 \log p_1$（cross-entropy，teacher 是 target）。

两个关键机制：
- **Sharpening**：teacher 用低温度，输出更尖锐（低熵）的分布
- **Centering**：$\tilde{z}_t = z_t - c$，$c \leftarrow mc + (1-m)\text{mean}(z_t)$，防止某个维度主导（防止 collapse 到单一 prototype）

**Multi-crop training**（PDF p59）：
- Teacher 只接收 **global crops**（2×224×224）
- Student 接收 **所有 crops**（2 global + N local 96×96）
- Local crops 强迫模型"从局部视角重建全局语义"→ local-to-global 一致性

**DINO + ViT 的特殊效果**（PDF p60）：
- ViT 的 patch token 结构使 attention 有空间可解释性
- `[CLS]` token 在 DINO 训练后，其 attention map **自然对齐 object boundaries**（emergent segmentation）
- ResNet 或有监督 ViT 无此效果——这是 self-distillation + ViT + multi-crop 三者共同作用的结果
- DINO ViT features 迁移到 detection/segmentation/retrieval 效果显著

<span style="color: #9b59b6">💡 **白话理解**：BYOL 证明了"不对称性就是防 collapse 的关键"——stop-gradient 让 teacher 成为一个固定的"目标"，student 追它但 teacher 不追 student，这个不对称性打破了梯度的平衡，防止两者坍缩到同一点。</span>

### CLIP：跨模态对比学习 `★★`

图像 encoder 和文本 encoder 同时训练，让匹配的（图，文）对的相似度最高。**CLIP 本质上是跨模态的 SimCLR**：batch 内匹配对为 positive，其他对为 negative。

**架构（dual-tower）**：
- Image encoder：ResNet-50 或 ViT（ViT-L/14 最强）
- Text encoder：Transformer（63M params）
- 两者投影到 **共享 512-dim 空间**，L2 normalize，相似度 = cosine
- 训练数据：400M (image, text) pairs（WIT 数据集），256 V100 GPUs，batch size **32768**

**Loss**（symmetric）：

$$L = -\log \frac{\exp(\text{sim}(v_i, t_i)/\tau)}{\sum_j \exp(\text{sim}(v_i, t_j)/\tau)}$$

Symmetric loss：image queries text **+** text queries image 两个方向都算。

**Zero-shot 分类**：把类别名字变成文本描述（"a photo of a {class}"），直接和图像比较相似度，不需要任何训练样本。

**CLIP 的重要性**（PDF p68）：
1. **Scalability**：互联网有数十亿图文对，无需人工标注
2. **Transfer without fine-tuning**：zero-shot CLIP 在许多数据集上匹敌有监督 linear probe；对 distribution shift 鲁棒（ImageNet robustness gap 下降 75%）
3. **Retrieval & grounding**：共享 embedding 空间支持双向图文检索
4. **Foundation model precursor**：CLIP 是 DALL-E 2、Stable Diffusion、GPT-4V、LLaVA 的视觉骨干

**CLIP 的局限**（PDF p69）：
1. **Prompt sensitivity**：措辞变化影响性能超过 10 个百分点，"a photo of a {class}" vs "{class}" 差异显著
2. **Data bias**：互联网图文对携带性别、种族、文化偏见
3. **Weak fine-grained grounding**：计数（"three dogs"）、空间关系（"left of"）、属性绑定（"red cube and blue sphere"）表现差
4. **Language quality ceiling**：噪声 alt-text 限制了表示质量

<span style="color: #9b59b6">💡 **白话理解**：CLIP 的 zero-shot 能力来自训练时图文对齐，不是"懂得推理"。它在 4 亿对图文数据上学会了"什么图和什么文字是一回事"，所以测试时直接拿类别名字当"文字探针"就能分类——但对细粒度任务（数数、空间关系）仍然力不从心。</span>

### SSL 方法全对比表（PDF p70）

| | AE | MAE | SimCLR | MoCo | BYOL | SimSiam | DINO | CLIP |
|--|--|--|--|--|--|--|--|--|
| **Supervision** | recon | mask | aug | aug | aug | aug | aug | img-txt |
| **Negatives** | none | none | batch | queue | none | none | none | batch |
| **Loss** | MSE | MSE | NCE | NCE | cos | cos | CE | CE |
| **Anti-collapse** | bottleneck | mask | neg | neg | EMA | sg+pred | centering | neg |
| **Backbone** | any | ViT | RN | RN | RN | RN | ViT | both |

**三句话总结**（PDF p70）：
- SimCLR learns **invariances**（what augmentations should not change）
- DINO learns **structure**（soft assignment to latent prototypes）
- CLIP learns **semantics**（aligns vision and language）

**5个 Final Takeaways**（PDF p71）：
1. **All methods design a supervision signal**：Reconstruction (AE, MAE) / Discrimination (SimCLR, MoCo) / Alignment (BYOL, SimSiam, DINO) / Language pairing (CLIP)
2. **"What should be the same" is the core design choice**：augmentation / masking / teacher-student / text pairing，各自定义不同的 invariance
3. **Know: objective, target, anti-collapse mechanism**（以及每种方法的 evaluation protocol）
4. **Representation learning = choosing the right invariance**：太多 invariance → 丢失有用信息；太少 → 学到 shortcuts；平衡是关键
5. **Asymmetry prevents collapse in non-contrastive methods**：Architecture（predictor）/ Optimization（stop-grad, EMA）/ Targets（centering, sharpening）

<span style="color: #e67e22">*自监督学习的目标是学到好的表示（representation）。生成模型的目标更激进：不只是理解数据，而是学会从头创造新数据——这需要对整个数据分布建模。*</span>

## Week 12 — 生成模型

### 核心目标

不是分类，而是**学习数据的分布** $p_\text{data}(x)$，从而能生成新样本。

### 1. GAN（生成对抗网络） `★★★`

> [!tip] 你训练 GAN 时，判别器 loss 很快降到接近零，但生成器 loss 剧烈震荡——从 JS 散度的角度解释，为什么训练好的判别器反而会"杀死"生成器的梯度？mode collapse 是另一个问题还是同一个问题？

两个网络的博弈：
- **Generator $G$**：把噪声 $z \sim p_z$ 映射到数据空间
- **Discriminator $D$**：判断输入是真实数据还是生成数据（输出 0-1 概率）

**Minimax 目标**：

$$\min_G \max_D \mathcal{L} = \mathbb{E}_{x \sim p_r}[\log D(x)] + \mathbb{E}_{z \sim p_z}[\log(1 - D(G(z)))]$$

**最优判别器**（固定 $G$ 时）：

$$D^*(x) = \frac{p_r(x)}{p_r(x) + p_g(x)}$$

**代入后**，GAN 本质上在最小化真实分布和生成分布的 **Jensen-Shannon 散度**：

$$\mathcal{L}(G, D^*) = 2\,\text{JS}(p_r \| p_g) - 2\log 2$$

**训练不稳定的原因**：当 $p_r$ 和 $p_g$ 的支撑集不重叠时（高维空间中常见），存在完美判别器，Generator 收到的梯度趋近于零。

**Mode Collapse**：Generator 只生成少数几种输出（学到了一个 mode），放弃覆盖整个数据分布。与梯度消失问题相互独立。

**DCGAN 架构（PDF p17-19）**：

| 组件 | 设计 |
|------|------|
| Generator | **Transposed convolution**（上采样），BatchNorm，ReLU，最后一层 **Tanh** |
| Discriminator | **Strided convolution**（下采样），BatchNorm（第一层除外），**LeakyReLU** |

> Tanh 让生成输出在 $[-1,1]$；LeakyReLU 防止梯度消失（避免 D 过强时 G 收不到信号）。

<span style="color: #9b59b6">💡 **白话理解**：GAN 就是"造假专家 vs 鉴定专家"的博弈。但有一个悖论：鉴定专家一旦太厉害（能完美区分真假），造假专家就完全收不到有用的反馈——就像考试卷子全是 0 分，根本不知道哪里错了。JS 散度在分布不重叠时是常数 $\log 2$，梯度为零。</span>

### 2. WGAN（Wasserstein GAN） `★★`

把 JS 散度替换为 **Wasserstein-1 距离**（Earth Mover's Distance），即使分布不重叠也有意义：

$$W(p_r, p_g) = \inf_{\gamma \in \Pi(p_r, p_g)} \mathbb{E}_{(x,y)\sim\gamma}[\|x - y\|]$$

对偶形式（Kantorovich-Rubinstein duality）：

$$W(p_r, p_g) = \sup_{\|f\|_L \leq 1} \mathbb{E}_{x \sim p_r}[f(x)] - \mathbb{E}_{x \sim p_g}[f(x)]$$

用 **Lipschitz 连续**的"critic"网络 $f_w$ 近似上确界：

**Critic loss**（最大化）：$\mathbb{E}_{x \sim p_r}[f_w(x)] - \mathbb{E}_{z \sim p_z}[f_w(G(z))]$

**Generator loss**（最小化）：$-\mathbb{E}_{z \sim p_z}[f_w(G(z))]$

Lipschitz 约束的实现：**Weight clipping** $w \leftarrow \text{clip}(w, -c, c)$（原始 WGAN）或 gradient penalty（WGAN-GP）。

**为什么 WGAN 更好 — JS vs KL vs W 比较**（PDF p34-36）：

以 $p_r = \mathcal{N}(0,1)$，$p_g = \mathcal{N}(\theta, 1)$，$\theta \neq 0$ 为例（非重叠情形）：

| 距离 | 值 | 含义 |
|------|-----|------|
| $D_{KL}(p_r \| p_g)$ | $+\infty$（当 $\theta \neq 0$） | 不可用 |
| $\text{JS}(p_r \| p_g)$ | $\log 2$（常数） | 梯度为零，Generator 无法学习 |
| $W(p_r, p_g)$ | $|\theta|$（连续，光滑）| 梯度为常数，始终有学习信号 |

→ **"Meaningful loss metric"**：WGAN 的 critic loss 与生成样本质量正相关（loss 降，样本质量升）。GAN 的 JS-based loss 与生成质量无相关性。

<span style="color: #9b59b6">💡 **白话理解**：Wasserstein 距离就是"把一堆沙子（$p_g$）搬成另一堆形状（$p_r$）的最小搬运代价"——即使两堆沙子完全不在同一个地方，也能算出一个有意义的距离。关键洞察：JS 散度在两个分布不重叠时是常数 $\log 2$，导数为零——Generator 完全收不到"往哪个方向走"的信号。W 距离则始终有梯度，训练稳定得多。WGAN 解决的是梯度消失，不是 mode collapse——两个是独立问题。</span>

### 3. VAE（变分自编码器） `★★★`

> [!tip] 你手写 VAE 时发现不能直接对 $z$ 采样然后反向传播——reparameterization trick 把随机性移到哪里？为什么这样可以让梯度流回 $\mu$ 和 $\sigma$？ELBO 里的 KL 项如果权重设为 0 会有什么问题？

**思路**：假设数据由隐变量 $z$ 生成，先验 $p(z) = \mathcal{N}(0,I)$，生成过程 $x \sim p_\theta(x|z)$。

直接最大化 $\log p_\theta(x)$ 是 intractable，引入近似后验 $q_\phi(z|x)$，推导 **ELBO**（Evidence Lower Bound）：

$$\log p_\theta(x) \geq \underbrace{\mathbb{E}_{z \sim q_\phi}[\log p_\theta(x|z)]}_{\text{重建项}} - \underbrace{D_\text{KL}(q_\phi(z|x) \| p(z))}_{\text{正则项}}$$

- **重建项**：Decoder 要能从 $z$ 重建 $x$
- **KL 正则项**：让近似后验靠近标准高斯先验，使隐空间连续可采样

**重参数化技巧（Reparameterization Trick）**：让采样步骤可以反向传播：

$$z = \mu_\phi(x) + \sigma_\phi(x) \odot \epsilon, \quad \epsilon \sim \mathcal{N}(0,I)$$

**重建 loss 的选择**（PDF p43）：
- **Binary Cross Entropy**：$\sum_i -[x_i \log \hat{x}_i + (1-x_i)\log(1-\hat{x}_i)]$（适用于 0/1 像素值，如 MNIST）
- **MSE**：$\sum_i (x_i - \hat{x}_i)^2$（适用于连续像素值）

**VAE 的缺点**：生成的图像偏模糊（重建 loss 倾向于平均多个模式）。

<span style="color: #9b59b6">💡 **白话理解**：VAE 说"每张图对应隐空间里的一个概率分布，不是一个点"。Reparameterization 是把随机性"外包"给 ε（没有参数），让梯度能绕过采样这个不可微的步骤，流回 μ 和 σ。KL 项强迫隐空间紧凑连续——否则隐空间会到处是空洞，采样出来全是垃圾。</span>

### 4. DDPM（扩散模型） `★★`

**思路**：学会逆转一个逐步加噪的过程。

**前向过程**（加噪，固定无参数）：

$$q(x_t | x_{t-1}) = \mathcal{N}\!\left(x_t; \sqrt{1-\beta_t}\,x_{t-1},\ \beta_t I\right)$$

直接从 $x_0$ 采样 $x_t$：

$$x_t = \sqrt{\bar{\alpha}_t}\,x_0 + \sqrt{1-\bar{\alpha}_t}\,\epsilon, \quad \epsilon \sim \mathcal{N}(0,I)$$

其中 $\bar{\alpha}_t = \prod_{s=1}^{t}(1-\beta_s)$。

**训练目标**（简化版，PDF p67）：让网络预测被加入的噪声：

$$\mathcal{L} = \mathbb{E}_{x_0, \epsilon, t}\!\left[\|\epsilon - \epsilon_\theta(\sqrt{\bar{\alpha}_t}x_0 + \sqrt{1-\bar{\alpha}_t}\epsilon,\, t)\|^2\right]$$

**反向过程参数化**（PDF p62）：学习去噪模型 $p_\theta(x_{t-1}|x_t)$：

$$p_\theta(x_{t-1}|x_t) = \mathcal{N}(x_{t-1};\, \mu_\theta(x_t,t),\, \Sigma_\theta(x_t,t))$$

网络预测噪声 $\epsilon_\theta$，通过 $\tilde{\mu}(x_t,x_0) = \frac{1}{\sqrt{\bar{\alpha}_t}}\!\left(x_t - \frac{1-\alpha_t}{\sqrt{1-\bar{\alpha}_t}}\epsilon_t\right)$ 得到去噪均值。

**Algorithm 1 — Training**（PDF p67）：
```
repeat:
  x0 ~ data
  t ~ Uniform(1,...,T)
  ε ~ N(0,I)
  Gradient step on ∇_θ ||ε - ε_θ(sqrt(ᾱ_t)x0 + sqrt(1-ᾱ_t)ε, t)||²
until converged
```

**Algorithm 2 — Sampling**（PDF p67）：
```
x_T ~ N(0,I)
for t = T,...,1:
  z ~ N(0,I) if t > 1, else z = 0
  x_{t-1} = 1/sqrt(α_t) * (x_t - (1-α_t)/sqrt(1-ᾱ_t) * ε_θ(x_t,t)) + σ_t * z
return x_0
```

**网络架构**：U-Net with ResNet blocks + self-attention；时间步 $t$ 用 sinusoidal positional embedding 编码，注入每个 ResNet block。

**生成时**：从高斯噪声 $x_T$ 出发，反复用网络预测噪声、去噪，逐步恢复 $x_0$（1000 步）。

<span style="color: #9b59b6">💡 **白话理解**：扩散模型就是学会"把一张图片磨成雪花点，然后反向播放这个过程"。前向加噪是固定的（不需要学），只需要学会逆向去噪。训练任务是"告诉我这张有噪声的图里加了多少噪声"——这比直接学生成分布简单得多。代价是推理需要 1000+ 步，比 GAN 慢很多。</span>

### 5. Flow Matching（新一代生成框架） `★`

**核心思想**：直接学习一个**速度场** $v_\theta(x,t)$，把噪声分布 $p_0$ 传输到数据分布 $p_1$。

训练数据构建：
$$x_0 \sim \mathcal{N}(0,I), \quad x_1 \sim p_\text{data}, \quad x_t = (1-t)x_0 + t x_1$$

插值点的目标速度（直线路径）：
$$u_t = x_1 - x_0$$

训练目标（监督回归）：
$$\mathcal{L}_\text{FM} = \mathbb{E}_{x_0, x_1, t}\!\left[\|v_\theta(x_t, t) - (x_1 - x_0)\|^2\right]$$

**生成**：从 $x_0 \sim \mathcal{N}(0,I)$ 出发，积分 ODE $\frac{dx}{dt} = v_\theta(x,t)$，得到 $x_1 \sim p_\text{data}$。

### 生成学习 Trilemma `★★★`

（PDF p70，Dhariwal & Nichol 2021）三个目标不能同时满足：

| 目标 | GAN | Diffusion | VAE |
|------|-----|-----------|-----|
| **高质量样本** | ✅ | ✅ | ⚠️ 偏模糊 |
| **快速采样** | ✅ | ❌ 1000 步 | ✅ |
| **多样性/Mode Coverage** | ❌ mode collapse | ✅ | ✅ |

→ **Flow Matching / Rectified Flow** 是解决 Diffusion 慢的新方向：直线路径 → 更少 ODE 步数（甚至 1 步）。

### 生成模型对比 `★★★`

| 模型 | 训练方式 | 能否计算密度 | 生成质量 | 代表缺点 |
|------|---------|------------|---------|---------|
| GAN | 对抗博弈 | ✗（隐式）| 高（清晰）| 训练不稳定，mode collapse |
| VAE | 最大化 ELBO | ✓（近似）| 中（模糊）| 生成图偏糊 |
| DDPM | 去噪回归 | ✗ | 高 | 推理 1000+ 步，慢 |
| Flow Matching | 速度场回归 | ✓（ODE）| 高 | 随机配对导致路径弯曲（Rectified Flow 改进）|

## 总结：四组的叙事地图

```
Group 1: 基础
  线性 → MLP → Backprop → Adam → Regularization

Group 2: 结构化数据
  MLP 忽略空间 → CNN (局部性+权重共享)
  CNN 需要网格 → GNN (邻居聚合)

Group 3: 序列数据
  静态 → RNN (记忆) → 梯度消失 → LSTM (门控)
  无法并行 → Transformer (Self-Attention)
  BERT (双向,理解) / GPT (因果,生成)
  图像序列化 → ViT
  O(N²) → SSM/Mamba

Group 4: 超越监督
  标签贵 → 自监督 (数据监督自己)
        → 对比学习 (SimCLR → BYOL → DINO)
        → 跨模态 (CLIP, MLLM)
        → 环境奖励 (RL → RLHF)
        → 学会生成 (GAN → VAE → Diffusion → Flow Matching)
```
