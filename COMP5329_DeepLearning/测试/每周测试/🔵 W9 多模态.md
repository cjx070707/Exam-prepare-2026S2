**Question 1.** *(MLLM Connector 设计 ★★★)* A team builds a multimodal assistant that takes as input a 448×448 image and a text query. Their current MLP-projection connector (LLaVA-style) produces 1,024 visual tokens, making the LLM context window too long for deployment. A teammate proposes switching to a Q-Former connector (BLIP-2-style). Which statement best explains the tradeoff?

- [ ] A. Q-Former reduces visual token count by using a fixed set of learnable query vectors that compress image features via cross-attention; the cost is that the queries are not conditioned on the text, so text-relevant fine-grained details may be lost
- [ ] B. Q-Former reduces visual token count by applying average pooling to the patch embeddings, trading spatial resolution for sequence length; MLP projection preserves spatial layout at the cost of more tokens
- [ ] C. Q-Former reduces visual token count because it applies causal masking to visual patches, selecting only the most attended patches; MLP projection attends to all patches uniformly
- [ ] D. Q-Former reduces visual token count by merging adjacent patch tokens with a learned merge gate; this is equivalent to Flamingo's cross-attention insertion approach but applied at the input stage

> [!note]- Answer
> **A**
>
> Q-Former 的核心机制：用 32 个（或固定数量的）可学习 query 向量，通过 cross-attention 从视觉 encoder 的特征中提炼信息。输出 token 数 = query 数（远小于 patch 数），实现压缩。代价是 query 不携带文本信息，可能漏掉对当前问题关键的细粒度视觉细节。
> - **B 错**：Q-Former 不是平均池化，是可学习 query 的 cross-attention 提炼
> - **C 错**：Q-Former 没有因果 mask 选 patch 的机制
> - **D 错**：Merge gate 是其他方法（如 PatchMerger）；与 Flamingo 的 cross-attention insertion 是完全不同类别（Flamingo 是 feature-level fusion，Q-Former 是 token-level fusion）

**Question 2.** *(LoRA 低秩分解 ★★★)* A team fine-tunes a 7B-parameter LLM (hidden dim $d=4096$) for customer service. They apply LoRA with rank $r=8$ to all query and value projection matrices. During an ablation, an intern suggests initializing both $A$ and $B$ with small random values $\sim \mathcal{N}(0, 0.01)$ instead of the standard $B=0, A \sim \mathcal{N}(0, \sigma^2)$. What is the most likely consequence?

- [ ] A. The random $B$ initialization causes $\Delta W = BA \neq 0$ at the start of training, meaning the model does not begin from the exact pre-trained weights and may diverge in the first few steps
- [ ] B. The random $B$ initialization causes the gradient magnitude of $A$ to be larger than normal, leading to an effective learning rate increase that destabilizes the LoRA update
- [ ] C. Both $A$ and $B$ being small-random means the low-rank update $\Delta W = BA$ has rank at most $r$ but the initial magnitude is too small to produce gradients, causing the LoRA parameters to remain near zero throughout training
- [ ] D. The random $B$ initialization changes the effective rank of the weight update from $r$ to $2r$, doubling the parameter count and negating the memory savings of LoRA

> [!note]- Answer
> **A**
>
> 标准初始化 $B=0$ 的精确意义：初始时 $\Delta W = BA = 0$，整个模型的输出与原始预训练模型**完全相同**，fine-tuning 从精确的预训练起点出发。若随机初始化 $B$，则训练开始时 $\Delta W \neq 0$，等价于在预训练权重基础上叠加了一个随机扰动——这会破坏预训练能力，尤其在前几步梯度还未指向有意义方向时容易发散。
> - **B 错**：梯度大小取决于损失，不是 B 的初始值
> - **C 错**：小但非零的 $BA$ 是有梯度的，不会导致梯度为零
> - **D 错**：秩是 $\min(\text{rank}(B), \text{rank}(A)) \leq r$，两个矩阵的乘积秩不超过 $r$

**Question 3.** *(MLLM 架构设计 ★★★)* A startup builds a visual QA system using a frozen CLIP-ViT-L/14 vision encoder, a trainable MLP connector, and a frozen LLaMA-2-7B backbone. During training, only the connector is updated. **(5 Marks)**

(a) 描述 MLP connector 在这个系统中的输入和输出：输入是什么形状的张量，输出接到 LLM 的哪个位置，训练时的监督信号来自哪里？**(2 Marks)**

(b) 团队发现模型在"数数"（如"图中有几只猫"）任务上表现很差。从 token-level fusion 架构的设计角度分析原因，并说明 feature-level fusion（如 Flamingo）能否缓解这一问题，为什么。**(2 Marks)**

(c) 若将 Connector 从 MLP 换为 Q-Former（32 个 learnable queries），推理时 LLM 的 context 长度如何变化？举出这一变化的一个好处和一个代价。**(1 Mark)**

> [!note]- Answer
> **(a)**
> - **输入**：CLIP-ViT 输出的 patch feature 序列，形状 $(N_\text{patch}, d_\text{vision})$，例如 ViT-L/14 处理 224×224 图像输出 $(196, 1024)$
> - **输出**：MLP 将每个 patch feature 线性投影到 LLM 的 token embedding 维度，输出形状 $(N_\text{patch}, d_\text{LLM})$，拼接到文本 token 序列前面，作为 LLM 的输入序列前缀
> - **监督信号**：LLM 的 next-token prediction loss（交叉熵），端到端从答案文本反传到 MLP 参数
>
> **(b)**
> MLP connector 是 token-level fusion——每个 patch feature 独立投影后拼接，没有跨 patch 的交互。视觉 encoder（CLIP-ViT）本身的 attention 权重是全局语义对齐的，不擅长精确计数（CLIP 训练目标是图文匹配，不是空间细粒度理解）。投影后的视觉 token 虽然都传入了 LLM，但 LLM 的 attention 读取这些 token 时会倾向于全局语义，难以精确定位每个个体。
>
> Flamingo 的 feature-level fusion（在 LLM 每层插入 cross-attention）让语言模型在每一层深度都能主动"查询"视觉特征——理论上有机会让 LLM 更细粒度地注意空间位置。但计数能力的根本瓶颈在于 vision encoder 的表示和训练数据，feature-level fusion 只是提高了视觉信息的注入深度，仍不能保证显著改善计数。
>
> **(c)**
> Q-Former 压缩到 32 个 token（vs MLP 的 196 token），context 长度大幅缩短。
> - **好处**：LLM 推理更快，显存占用更低，可处理更长文本
> - **代价**：32 个 query 将整张图压缩为 32 个 token，细粒度空间信息被压缩丢失，对需要精确位置理解的任务（如 OCR、计数）表现更差

**Question 4.** *(LoRA + MoE 效率机制 ★★★)* A lab deploys a 70B-parameter LLM on a cluster with limited GPU memory. They evaluate two efficiency techniques: LoRA (rank $r=16$) applied to all attention weight matrices, and replacing dense FFN layers with MoE layers ($E=8$ experts, top-$k=2$ activated per token). **(4 Marks)**

(a) LoRA 的参数量节省来自哪里？给定注意力矩阵 $W_Q \in \mathbb{R}^{4096 \times 4096}$，LoRA $r=16$ 时，$W_Q$ 对应的可训练参数数量是多少？与全参数微调相比节省了多少倍？**(2 Marks)**

(b) MoE 层替换 FFN 层后，理论上每个 token 的计算量（FLOPs）与等规模 Dense FFN 相比如何？为什么总参数多但推理成本可以不成比例地低？**(2 Marks)**

> [!note]- Answer
> **(a)**
> LoRA 冻结原始权重 $W_0$，只训练低秩分解 $B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times d}$。
>
> $W_Q$ 对应的 LoRA 可训练参数 = $d \times r + r \times d = 2 \times 4096 \times 16 = 131{,}072$
>
> 全参数微调的参数量 = $4096 \times 4096 = 16{,}777{,}216$
>
> 节省倍数 = $16{,}777{,}216 / 131{,}072 = \mathbf{128}$ 倍（即 $d/r = 4096/16 = 256$，两侧各有一个矩阵，所以 $d/(r) = 256$ 对每个矩阵的一侧成立，总节省为 $d \cdot d / (2dr) = d/(2r) = 128$ 倍）
>
> **(b)**
> MoE 每个 token 只激活 top-$k=2$ 个专家，计算量 ≈ 2 个 FFN 的 FLOPs，与总专家数 $E=8$ 无关。
>
> 总参数量 ∝ $E$（8 个专家各有完整权重），但每 token 计算量 ∝ $k=2$（只前向经过 2 个专家）。对比等计算量的 Dense FFN：若 MoE 每专家大小与 Dense FFN 相同，则 MoE 有 8× 参数但只用 2×/8× = 1/4 的计算量（或者说同计算量下 MoE 有 4× 更大的容量）。
>
> 直觉：总参数多 → 模型容量大、性能好；每 token 计算少 → 推理速度快、显存带宽需求低。MoE 用路由（gating）在训练和推理中解耦了"容量"和"计算"。
