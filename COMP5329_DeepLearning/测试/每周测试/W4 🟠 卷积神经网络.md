**Question 1.** *(输出尺寸公式 ★★★)* A digit classifier takes 28×28 grayscale inputs. A conv layer applies a 5×5 kernel with stride 1 and no padding. What is the output spatial size?

- [ ] A. 22×22
- [ ] B. 23×23
- [x] C. 24×24
- [ ] D. 25×25

> [!note]- Answer
> **C**
>
> O = (28 + 0 − 5)/1 + 1 = **24**
> - **A 错**：误用 N − K − 1 + 1
> - **B 错**：忘记 +1，只算了分子 23
> - **D 错**：多加了 1，算成 N − K + 2

**Question 2.** *(ResNet 退化问题 ★★★)* A 56-layer plain CNN shows higher training error than a 20-layer version on the same data. Validation error follows the same pattern, ruling out overfitting. What best explains this?

- [ ] A. Stacking too many layers reduces the effective learning rate across the network, preventing the optimizer from converging to a low-loss region
- [ ] B. The deeper model saturates representational capacity after a certain depth, causing earlier activations to be overwritten as depth increases
- [x] C. The 56-layer network should theoretically match the 20-layer one by learning identity mappings across extra layers, but optimization fails to find this solution
- [ ] D. Excessive depth amplifies gradient noise during SGD, causing the 56-layer model to oscillate around a suboptimal loss basin rather than converge

> [!note]- Answer
> **C**
>
> 退化问题（degradation problem）：更深的网络理论上包含更浅网络（令多余层学恒等映射），但梯度优化无法找到这个解，训练误差本身随深度上升。这是**优化难度**问题，不是容量问题，也不是过拟合。
> - **A/D 错**：把问题归因于 LR 或梯度噪声，不是退化问题的定义
> - **B 错**："saturation of capacity" 不是退化问题的机制

**Question 3.** *(Inductive Bias CNN vs MLP ★★★)* An MLP and a CNN are trained on an image dataset with equal parameter budgets. The CNN achieves lower test error despite identical training error. Which factor most directly accounts for this?

- [ ] A. CNNs apply pooling layers that reduce spatial resolution, discarding high-frequency noise that misleads the MLP's fully-connected layers
- [x] B. CNNs encode structural assumptions about images — local feature relevance and position consistency — that reduce the effective hypothesis space
- [ ] C. CNNs stack more non-linear activations per parameter, giving them greater expressive power relative to the MLP at the same parameter count
- [ ] D. CNNs implicitly regularise through parameter sharing in a way that penalises large weights, analogous to the effect of explicit weight decay

> [!note]- Answer
> **B**
>
> CNN 的 inductive bias：局部连接（local connectivity）+ 权重共享（weight sharing）+ 平移等变性（translation equivariance）。这些先验与图像数据的真实结构匹配，有效压缩了假设空间。
> - **A 错**：Pooling 的核心是平移不变性，不是"去噪"
> - **C 错**：非线性层数多不是 inductive bias 的来源
> - **D 错**：权重共享确实有正则化效果，但 B 是更根本的解释（结构先验匹配，不只是正则化）

**Question 4.** *(1×1 Conv Bottleneck ★★)* A ResNet bottleneck block processes 256-channel feature maps using a 1×1→3×3→1×1 sequence, compressing to 64 channels before the 3×3 layer. Why does this reduce computation compared to two 3×3 layers on 256 channels?

- [ ] A. The 1×1 layers reduce channel count before the 3×3 conv and restore it after, so the expensive 3×3 operates on 64 channels instead of 256
- [ ] B. The 1×1 convolutions act as residual connections within the bottleneck, bypassing the 3×3 computation for the majority of channels
- [ ] C. A 3×3 conv on 64 channels has the same multiply-add cost as on 256 channels; the saving comes from having fewer output feature maps at the block's exit
- [ ] D. The 1×1 layers apply channel-wise normalisation that eliminates the need for batch normalisation inside the 3×3 layer, reducing total operations

> [!note]- Answer
> **A**
>
> 1×1(256→64)：256 mults/position；3×3(64→64)：9×64=576 mults/position；1×1(64→256)：64 mults/position。Total ≈ **896**。
> 两个 3×3(256→256)：9×256×2=**4608**。节省约 5×。
> - **B 错**：1×1 不是 skip connection，skip connection 是整个 block 外层的 H(x)=F(x)+x
> - **C 错**：3×3 的计算量确实随 input channels 线性增长
> - **D 错**：BN 不是此处的解释

**Question 5.** *(Dilated Conv ★★)* A segmentation model needs to capture features across a 9×9 region. The team uses a 3×3 kernel with dilation rate 4 instead of a standard 9×9 kernel. Which statement accurately characterises this tradeoff?

- [ ] A. Both achieve a 9×9 effective receptive field; the dilated 3×3 uses 9 trainable weights and samples 9 positions within the region, while the 9×9 kernel uses 81 weights and covers all 81 positions
- [ ] B. The dilated 3×3 achieves a 7×7 effective receptive field with 9 parameters; the 9×9 kernel covers more pixels but at higher parameter cost
- [ ] C. Both achieve the same 9×9 receptive field and process identical pixel sets; the dilated kernel reduces computation only by producing a smaller output feature map
- [ ] D. The dilated 3×3 uses 9 parameters like a standard 3×3 but runs faster because dilation skips the arithmetic at intermediate pixel positions

> [!note]- Answer
> **A**
>
> Effective receptive field = K + (K−1)(d−1) = 3 + 2×3 = **9×9**。参数数 = 3×3 = 9（vs 9×9 kernel 的 81）。但 dilated conv 在 9×9 区域内只采样 9 个等间距位置（每隔 3 个像素取一个），不覆盖全部 81 个像素。
> - **B 错**：d=4 对应 9×9 感受野，不是 7×7
> - **C 错**：dilated conv 确实跳过了中间像素，两者覆盖的像素集合不同
> - **D 错**："跳过算术"描述不准确，dilated conv 仍然计算 9 次乘加

**Question 6.** *(ResNet 综合 ★★★)* A computer vision team replaces a plain 50-layer CNN with ResNet-50. **(7 Marks)**

(a) Define the degradation problem. Why does training error rise with depth in plain CNNs, and why does this differ from overfitting? **(3 Marks)**

(b) Write the residual block formula. What gradient property does the skip connection provide during backpropagation? **(2 Marks)**

(c) A bottleneck block: 1×1(256→64) → 3×3(64→64) → 1×1(64→256). At a single spatial position, compare its multiply-adds to two 3×3(256→256) layers. **(2 Marks)**

> [!note]- Answer
> **(a)**
> 退化问题：随深度增加，**训练误差**（而非测试误差）本身上升。理论上 56 层网络至少可以通过令多余层学习恒等映射来匹配 20 层性能，但梯度优化无法找到这个解——这是**优化难度**问题，不是容量问题。
>
> 区别于过拟合：过拟合表现为训练误差低、测试误差高；退化问题是训练误差也更高，说明深层网络在拟合训练集上就已经失败了。
>
> **(b)**
> $$H(x) = F(x) + x$$
>
> 其中 F(x) 是两层卷积的残差，x 是 skip connection（恒等映射）。
>
> 梯度优势：反向传播时 $\frac{\partial H}{\partial x} = \frac{\partial F}{\partial x} + 1$，"+1" 保证即使 F 的梯度趋近 0，梯度仍有至少 1 的量流回浅层，有效缓解梯度消失。
>
> **(c)**
> 两个 3×3(256→256)：每层 3×3×256=2304，两层共 **4608** multiply-adds。
>
> Bottleneck：1×1×256=256 + 3×3×64=576 + 1×1×64=64 = **896** multiply-adds。
>
> 节省约 **5×**，同时输入输出维度保持 256 不变（便于 skip connection 直接相加）。

**Question 7.** *(Conv Forward ★★★)* Implement a 2D convolution forward pass without any framework conv op. Single channel input, single filter, no padding. **(6 Marks)**

```python
def conv2d_forward(x, kernel, stride=1):
    """
    x:      (H_in, W_in) — input
    kernel: (K, K)       — filter
    stride: int
    returns: (H_out, W_out)
    """
    ...
```

**(2 Marks)** Compute `H_out` and `W_out` using the correct formula

**(2 Marks)** Initialise output array and write the 4-loop structure (h_out, w_out, kh, kw)

**(2 Marks)** Write the inner accumulation correctly

> [!note]- Answer
> ```python
> def conv2d_forward(x, kernel, stride=1):
>     H_in, W_in = x.shape
>     K = kernel.shape[0]
>     H_out = (H_in - K) // stride + 1
>     W_out = (W_in - K) // stride + 1
>
>     out = torch.zeros(H_out, W_out)
>     for i in range(H_out):
>         for j in range(W_out):
>             for kh in range(K):
>                 for kw in range(K):
>                     out[i, j] += x[i*stride + kh, j*stride + kw] * kernel[kh, kw]
>     return out
> ```
>
> **关键点**：
> - 输出尺寸用 `//`（整除），对应 P=0 的标准公式
> - 4 层循环：输出位置 (i,j) × kernel 位置 (kh,kw)
> - 输入索引：`i*stride + kh`，stride 控制采样步长
> - 忘记 stride 乘法或输出尺寸公式错误各扣 2 分
