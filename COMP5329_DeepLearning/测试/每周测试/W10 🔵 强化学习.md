**Question 1.** *(DQN 训练不稳定 ★★★)* A research team trains a DQN agent from raw pixel input to play a video game. They use a single neural network without Experience Replay or Target Network, and find that training diverges after a few thousand steps. A teammate proposes adding Experience Replay. A second teammate instead proposes a Target Network. Which statement correctly distinguishes the problem each trick solves?

- [ ] A. Experience Replay breaks temporal correlation between consecutive transitions by sampling uniformly from a replay buffer; Target Network stabilizes training by fixing the Q-value target for a number of steps, decoupling it from the rapidly changing online network
- [ ] B. Experience Replay increases sample efficiency by reusing transitions multiple times; Target Network prevents overfitting by regularizing the Q-network weights toward the initial random values
- [ ] C. Experience Replay prevents the non-stationary target problem by maintaining a moving average of Q-values; Target Network solves the correlation problem by generating diverse actions using ε-greedy
- [ ] D. Experience Replay reduces the action space by grouping similar transitions; Target Network eliminates the need for a discount factor γ by normalizing the Bellman target

> [!note]- Answer
> **A**
>
> DQN 不稳定的两个根本原因（PDF p20）：
> - **Correlations between samples**：连续时间步的 transition 强相关，违反 i.i.d.，梯度估计有偏 → Experience Replay 随机采样打破相关性
> - **Non-stationary targets**：target $r + \gamma\max Q_w(s',a')$ 里的 $w$ 和被优化的 $w$ 是同一个，追移动靶 → Target Network 固定 N 步后才同步
> - **B 错**：Replay 解决相关性，不是防过拟合；Target Network 和正则化无关
> - **C 错**：两者的作用完全对调了
> - **D 错**：均为错误描述

**Question 2.** *(DPO vs RLHF ★★★)* A startup fine-tunes a 13B LLM for customer service. Their infrastructure team warns that loading 4 large models simultaneously will exceed GPU memory. The ML team proposes switching from RLHF (PPO) to DPO. Which statement correctly explains how DPO reduces memory requirements while still optimizing the same underlying objective?

- [ ] A. DPO eliminates the Reward Model by showing that the optimal policy under the KL-constrained RLHF objective can express the reward implicitly as a log-ratio of policy to reference; comparing two responses cancels the intractable partition function Z(x)
- [ ] B. DPO reduces memory by replacing the Value Model (critic) with group statistics computed from multiple sampled outputs, requiring only the Policy and Reward models in memory
- [ ] C. DPO eliminates the Reference Model by using the SFT model directly as the reward signal; the KL penalty is approximated by a hinge-loss margin between preferred and rejected responses
- [ ] D. DPO reduces memory by distilling the four PPO models into a single model using knowledge distillation before fine-tuning on preference data

> [!note]- Answer
> **A**
>
> DPO 的核心推导（PDF p36-37）：KL 约束的 RLHF 目标有闭式最优策略 $\pi^*(y|x) = \frac{1}{Z(x)}\pi_\text{ref}(y|x)\exp(r/\beta)$，反解出 $r(x,y) = \beta\log\frac{\pi^*(y|x)}{\pi_\text{ref}(y|x)} + \beta\log Z(x)$。比较两个回复时 $Z(x)$ 约掉，reward gap 变成 policy log-ratio gap，无需单独训练 RM。只需 $\pi_\theta$ 和 $\pi_\text{ref}$ 两个模型。
> - **B 错**：描述的是 GRPO（用 group statistics 替代 Value Model）
> - **C 错**：DPO 仍然需要 Reference Model，KL 惩罚是通过 log-ratio 保留的
> - **D 错**：DPO 不涉及知识蒸馏

**Question 3.** *(DQN 架构设计 ★★★)* A robotics team builds a DQN agent to control a robot arm. The state space is the raw camera image (4 stacked frames of 84×84 pixels), and there are 6 discrete actions. **(5 Marks)**

(a) 描述 Q-Network 的输入和输出结构：相比"输入 (s,a) 输出单个 Q 值"的设计，为什么 DQN 选择"输入 s 输出所有动作的 Q 值"？**(2 Marks)**

(b) 写出 DQN 的训练 Loss 公式，标注哪一部分是 Target（目标值），解释 DQN 不加 Target Network 时为什么会出现训练不稳定？**(2 Marks)**

(c) Experience Replay buffer 存储的每条记录是什么格式？从 buffer 中**均匀**采样（而不是按时序顺序采样）的目的是什么？**(1 Mark)**

> [!note]- Answer
> **(a)**
> **输入 s 输出所有动作 Q 值**（正确设计 ✅）：只需一次前向，同时得到所有 $m$ 个动作的 Q 值，可以直接取 $\arg\max$ 选动作。
>
> **输入 (s,a) 输出单个 Q 值**（错误设计 ❌）：每次选动作都要对所有 $m$ 个动作各跑一次网络，推理成本是前者的 $m$ 倍。
>
> **(b)**
> DQN Loss（MSE on Bellman target）：
>
> $$l = \left(\underbrace{r + \gamma \max_{a'} Q_w(s', a')}_{\text{Target}} - Q_w(s,a)\right)^2$$
>
> 不加 Target Network 时，target 中的 $Q_w(s',a')$ 和被优化的 $Q_w(s,a)$ 使用**同一组参数** $w$。每次梯度更新都会改变 $w$，target 随之漂移——像追一个不停移动的靶子，容易震荡发散（non-stationary target 问题）。
>
> Target Network 的解决方案：用一个慢更新的参数副本 $w^-$ 计算 target，$w^-$ 每隔 $C$ 步才从 $w$ 同步一次，训练期间 target 相对固定。
>
> **(c)**
> 格式：$(s_t, a_t, r_{t+1}, s_{t+1})$（当前状态、动作、奖励、下一状态）
>
> 均匀采样的目的：打破时序相关性（temporal correlation）。连续时间步的 transition 非常相似（$s_t$ 和 $s_{t+1}$ 几乎相同），若按顺序采样，梯度估计会有偏；均匀随机采样使得每个 mini-batch 的样本近似 i.i.d.，满足 SGD 的独立性假设，稳定训练。

**Question 4.** *(RLHF / DPO / GRPO 对比 ★★★)* A lab aligns a 7B-parameter LLM using human preference data. Three engineers propose three different methods: (1) RLHF with PPO, (2) DPO, and (3) GRPO. **(6 Marks)**

(a) RLHF Stage 2 训练的 Reward Model 用的是 Bradley-Terry 模型。写出 RM 的损失函数 $\mathcal{L}_\text{RM}(\phi)$，解释其中每个符号的含义。**(2 Marks)**

(b) RLHF Stage 3 的目标函数包含一个 KL 惩罚项。写出完整的 RLHF Objective，解释为什么必须加 KL 惩罚，以及 $\beta$ 越大意味着什么。**(2 Marks)**

(c) GRPO 用"组统计"替代了 Critic/Value Model。写出 GRPO 的 Group-Relative Advantage 公式 $\hat{A}_i$，并说明：与 PPO 相比，GRPO 节省了哪个模型的显存？代价是什么？**(2 Marks)**

> [!note]- Answer
> **(a)**
> Bradley-Terry 模型建模人类偏好概率：
>
> $$P(y_w \succ y_l \mid x) = \sigma\!\left(r_\phi(x, y_w) - r_\phi(x, y_l)\right)$$
>
> RM 损失（最大似然训练）：
>
> $$\mathcal{L}_\text{RM}(\phi) = -\mathbb{E}_{(x,y_w,y_l)}\!\left[\log \sigma\!\left(r_\phi(x,y_w) - r_\phi(x,y_l)\right)\right]$$
>
> - $\phi$：Reward Model 参数
> - $y_w$：人类标注为"胜出"的回复（winner）
> - $y_l$：人类标注为"落败"的回复（loser）
> - $\sigma$：sigmoid 函数
> - 训练目标：$r_\phi(y_w) > r_\phi(y_l)$，gap 越大越好，使偏好概率趋近于 1
>
> **(b)**
>
> $$\max_\theta\, \mathbb{E}_{x,\, y \sim \pi_\theta}\!\left[r_\phi(x, y) - \beta \cdot D_\text{KL}\!\left(\pi_\theta \| \pi_\text{ref}\right)\right]$$
>
> **必须加 KL 惩罚的原因**：防止 **reward hacking**——策略为了让 RM 打高分而生成奇怪输出（如重复、无意义的高分内容），这些内容在 RM 训练分布之外，RM 无法正确评价。KL 惩罚让策略与 SFT 参考模型保持接近，维持生成质量。
>
> **$\beta$ 越大**：对漂移惩罚越重，策略越靠近 $\pi_\text{ref}$（更保守，reward 可能偏低，但输出更安全稳定）。$\beta$ 越小：允许更大漂移，reward 可能更高，但 reward hacking 风险更大。
>
> **(c)**
> GRPO Group-Relative Advantage（对每个 prompt 采样 $G$ 个回复，用组内统计归一化）：
>
> $$\hat{A}_i = \frac{r_i - \text{mean}(\{r_1, \ldots, r_G\})}{\text{std}(\{r_1, \ldots, r_G\})}$$
>
> - **节省**：PPO 需要 4 个模型（Policy + Reference + Reward + Value/Critic），GRPO 用组统计替代 Value Model，只需 3 个模型。对 7B 模型，节省约 7B 参数的显存。
>
> - **代价**：每次更新需要对每个 prompt 采样 $G=16\sim64$ 个回复，**推理开销增大**（需额外的 forward passes）。但相比训练一个全尺寸 Critic 网络，整体仍是更经济的选择。
