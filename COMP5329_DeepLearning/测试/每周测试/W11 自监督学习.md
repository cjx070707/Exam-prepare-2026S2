**Question 1.** *(SimSiam vs BYOL ★★★)* A research team is training a self-supervised vision model on a dataset of 500,000 unlabeled medical images. They cannot afford the memory overhead of a separate momentum encoder copy. They test both BYOL and SimSiam. An intern removes the predictor head from SimSiam (to simplify the architecture) while keeping stop-gradient. Which statement is most accurate?

- [ ] A. Removing the predictor from SimSiam will still work because stop-gradient alone is sufficient to prevent collapse; the predictor only marginally improves linear probe accuracy
- [ ] B. Removing the predictor from SimSiam causes collapse even with stop-gradient, because the predictor introduces the architectural asymmetry that prevents both branches from converging to the same constant representation
- [ ] C. SimSiam without a predictor is equivalent to BYOL without EMA, so both will collapse at the same rate; the EMA in BYOL is the sole anti-collapse mechanism
- [ ] D. Removing the predictor from SimSiam is beneficial when the dataset is small, because the predictor tends to overfit the augmentation policy rather than the semantic content

> [!note]- Answer
> **B**
>
> PDF p43 的 ablation 表明：SimSiam 中 stop-gradient 和 predictor **都是必须的**。
> - 无 stop-gradient → collapse（loss 平在 -1，两分支输出常数向量，output std → 0）
> - 有 stop-gradient 但无 predictor → 同样 collapse
> - 两者均有 → 正常学习（kNN acc 上升至 50+）
>
> 直觉：stop-gradient 让一个分支变成"固定目标"，predictor 通过引入不对称的映射（Branch 1: encoder + projector + predictor；Branch 2: encoder + projector + stop-grad）打破了两分支的对称性。去掉 predictor 后，两分支完全对称，stop-grad 虽然阻止梯度流动，但没有了预测任务的不对称性，表示仍会坍缩。
> - **A 错**：stop-grad alone ≠ anti-collapse（PDF ablation 明确反驳）
> - **C 错**：BYOL 的 EMA 是主要机制，不是"唯一"；两者 collapse 机制不同
> - **D 错**：与 dataset size 无关，是架构设计问题

**Question 2.** *(DINO centering ★★★)* A team implements DINO on a ViT-Small backbone. During training they observe that the teacher's softmax output always peaks at the same few prototype dimensions regardless of the input image — the model has collapsed to a dominant prototype. A teammate suggests fixing this by removing centering. Another suggests removing sharpening. Which response correctly diagnoses the issue?

- [ ] A. Centering prevents collapse to a dominant prototype by subtracting a running mean of teacher outputs; removing centering would worsen the collapse. Sharpening (low temperature) makes the teacher's distribution sharp but does not cause dominant-prototype collapse
- [ ] B. Sharpening causes the dominant prototype issue by forcing teacher outputs to be low-entropy; removing sharpening will diversify the teacher's distribution and fix the collapse. Centering is unnecessary if temperature is properly tuned
- [ ] C. Both centering and sharpening contribute equally to dominant-prototype collapse; the correct fix is to increase the student's temperature to match the teacher's, making both distributions flat
- [ ] D. The dominant prototype issue is caused by the EMA coefficient being too large (close to 1.0), making the teacher update too slowly; neither centering nor sharpening is relevant

> [!note]- Answer
> **A**
>
> DINO 的两个关键机制作用不同（PDF p53, p58）：
> - **Centering**：$\tilde{z}_t = z_t - c$，$c \leftarrow mc + (1-m)\text{mean}(z_t)$。从 teacher 输出减去 batch 均值，防止某个维度长期主导 → 防止 dominant prototype collapse。**移除 centering 会让问题更严重**。
> - **Sharpening**：teacher 用低温度，让输出分布更尖锐（低熵）→ 给 student 提供更清晰的学习目标，防止 mode averaging collapse（student 学到全均匀分布）。它**不会**导致 dominant prototype 问题。
>
> 题目描述的是 dominant prototype collapse（所有输入都映射到同一个 prototype），对应的修复是确保 centering 正常工作，不是移除它。
> - **B 错**：sharpening 不是 dominant prototype 的原因
> - **C 错**：两者作用独立，不是"均等贡献"
> - **D 错**：EMA 系数影响 teacher 更新速度，不直接导致 dominant prototype collapse

**Question 3.** *(InfoNCE 与对比学习 ★★★)* A team trains a contrastive learning system on 1M unlabeled satellite images. They use InfoNCE loss with cosine similarity. During a debugging session, they find the loss converges quickly but linear probe accuracy on a downstream land-cover classification task is poor. **(5 Marks)**

(a) 写出 InfoNCE Loss 的完整公式，标注 $q$, $k^+$, $k_j$, $\tau$ 各代表什么。写出 InfoNCE 对 $q$ 的梯度形式，解释为什么"难负样本"（和 $q$ 相似的负样本）获得更大的梯度权重。**(2 Marks)**

(b) 温度参数 $\tau$ 过大（如 $\tau = 2.0$）和过小（如 $\tau = 0.01$）分别有什么问题？说明各自导致的训练失败模式。**(2 Marks)**

(c) 团队发现所有图像都使用同一种数据增强（只做随机水平翻转）。从"invariance 的设计"角度分析：这样的增强会导致模型学到什么样的不变性，为什么这对下游土地覆盖分类任务不够用？**(1 Mark)**

> [!note]- Answer
> **(a)**
>
> $$\mathcal{L} = -\log \frac{\exp(q \cdot k^+ / \tau)}{\sum_j \exp(q \cdot k_j / \tau)}$$
>
> - $q$：当前图像某次增强后的 query 表示
> - $k^+$：同一图像另一种增强的 key 表示（正样本）
> - $k_j$：batch 内所有 key（含正样本和负样本）
> - $\tau$：温度参数，控制分布的尖锐程度
>
> 梯度形式（对 $q$ 的方向）：
> $$\frac{\partial \mathcal{L}}{\partial q} = -\frac{1}{\tau}k^+ + \frac{1}{\tau}\sum_j p_j k_j, \quad p_j = \frac{\exp(q \cdot k_j/\tau)}{\sum_l \exp(q \cdot k_l/\tau)}$$
>
> 权重 $p_j$ 正比于 $q$ 和 $k_j$ 的相似度。难负样本（和 $q$ 高度相似的负样本）$q \cdot k_j$ 大 → $p_j$ 大 → 梯度中该方向权重更大。直觉：模型最容易"看走眼"的负样本提供最有价值的学习信号。
>
> **(b)**
>
> **$\tau$ 过大**（如 2.0）：softmax 变平，所有负样本的权重 $p_j$ 接近均匀。模型无法区分难负样本和易负样本，梯度信号过弱，学习缓慢；表示空间中 positive 和 negative 的分离力不足，最终学到的表示不够有判别性。
>
> **$\tau$ 过小**（如 0.01）：softmax 极度尖锐，梯度几乎全部来自一个最大的负样本，其他负样本贡献极小。如果这个"最难"的负样本恰好是假负样本（false negative：实际上同类但被当作负样本），则模型会被错误梯度主导，产生错误的分离。训练不稳定。
>
> **(c)**
>
> 只用水平翻转作为增强，模型学到的不变性是"水平翻转后表示相同"。对卫星图像的土地覆盖分类，判别性特征在于**颜色、纹理和多尺度空间结构**（如农田的纹理 vs 水体的颜色 vs 城市的边缘）。水平翻转保留了所有这些特征，模型没有被迫学习对颜色扰动、遮挡、尺度变化等鲁棒的语义表示。好的增强策略（如 color jitter + random crop + blur）应该迫使模型丢弃非语义特征、保留语义特征，从而学到对下游任务有用的不变性。

**Question 4.** *(DINO / CLIP / SSL 方法设计对比 ★★★)* A startup wants to build a foundation model for medical imaging (X-rays). They compare three approaches: (1) DINO with ViT-Base, (2) SimCLR with ResNet-50, and (3) CLIP trained on 10M radiology report–image pairs. **(6 Marks)**

(a) DINO 和 SimCLR 都可以学到视觉表示，但它们的监督信号来源和 anti-collapse 机制完全不同。完成下表，说明两者在以下维度的区别。**(2 Marks)**

| 维度 | DINO | SimCLR |
|------|------|--------|
| 监督信号（supervision） | | |
| 是否需要负样本 | | |
| Loss 类型 | | |
| Anti-collapse 机制 | | |

(b) 若用 DINO + ViT 训练，团队发现训练完成后无需任何标签，ViT 的 attention map 已经能定位 X-ray 中的肺部区域。从 DINO 的设计出发解释这一"emergent segmentation"现象的来源：哪三个组件共同产生了这一效果？**(2 Marks)**

(c) 相比 DINO，CLIP 提供了什么额外的能力？说明 CLIP zero-shot 分类的原理。同时给出 CLIP 在医疗影像场景下的一个具体局限，并解释其原因。**(2 Marks)**

> [!note]- Answer
> **(a)**
>
> | 维度 | DINO | SimCLR |
> |------|------|--------|
> | 监督信号 | 数据增强（augmentation-based self-distillation）；student 预测 teacher 的 softmax 分布 | 数据增强；让同图两个增强的表示靠近，不同图分开 |
> | 是否需要负样本 | **否**（non-contrastive） | **是**（batch 内其他图像为负样本，需大 batch） |
> | Loss 类型 | **Cross-entropy**（分布匹配：$-p_t \log p_s$） | **InfoNCE**（$-\log\frac{\exp(q\cdot k^+/\tau)}{\sum\exp(q\cdot k_j/\tau)}$） |
> | Anti-collapse 机制 | **Centering**（防 dominant prototype）+ **Sharpening**（防 mode averaging）+ EMA teacher | **Negative samples**（batch 内负样本排斥，防止坍缩到单点） |
>
> **(b)**
>
> DINO + ViT 产生 emergent object segmentation 的三个必要组件（PDF p60）：
>
> 1. **Self-distillation（DINO 目标本身）**：让 student 从不同视角（local crops）重建 teacher 看到的全局语义分布，驱动模型学习 local-to-global 一致性，即"局部 patch 的表示需要反映全局语义结构"
>
> 2. **ViT 的 token-based 结构**：ViT 将图像切成 patch token 序列，self-attention 在 patch 之间直接交互，attention 权重具有空间可解释性。`[CLS]` token 通过 attention 聚合所有 patch 的信息，自然形成"对哪些 patch 关注"的空间分布
>
> 3. **Multi-crop training**：teacher 只处理全局视图，student 处理 local 小 crop，迫使模型从局部观察推断全局结构。这个 local-to-global 压力让 [CLS] 的 attention 学会"从局部 patch 辨识语义上重要的区域"
>
> 三者缺一不可（ResNet 无此效果，有监督 ViT 亦无此效果）。
>
> **(c)**
>
> **CLIP 的额外能力**：CLIP 将视觉和语言对齐到同一个 embedding 空间，支持**语言驱动的 zero-shot 分类和图文检索**，而 DINO 只能学习视觉内部结构。
>
> **Zero-shot 分类原理**：把每个类别名转成文本 prompt（如 "a chest X-ray showing pneumonia"），用 text encoder 编码为文本 embedding。测试时将 X-ray 图像用 image encoder 编码，计算与所有类别文本 embedding 的 cosine similarity，取最高的作为预测类别。不需要任何标注训练样本。
>
> **医疗场景的具体局限**：CLIP 存在 **weak fine-grained grounding** 和 **language quality ceiling**。放射科报告通常描述细粒度的解剖位置关系（"右上叶出现 3mm 结节"）和属性绑定，而 CLIP 只匹配全局图文语义，无法精确定位空间关系。此外，CLIP 训练数据的文本描述质量上限决定了模型能力上限——若医疗文本噪声多或描述不准确，模型无法学到准确的临床特征对应。
