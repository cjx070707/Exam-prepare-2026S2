**Question 1.** *(L1 稀疏性 ★★★)* A model predicts treatment outcomes from 10,000 gene features, but domain experts estimate only ~50 genes are relevant. The team wants regularisation to automatically zero out irrelevant features. Which strategy is most appropriate?

- [ ] A. L2, because it shrinks all weights uniformly and drives irrelevant features toward zero over many epochs
- [x] B. L1, because its diamond-shaped constraint region tends to produce solutions where irrelevant weights are exactly zero
- [ ] C. L2, because its smooth constraint surface makes optimisation more stable when most features are irrelevant
- [ ] D. L1, because the absolute-value gradient is larger than the squared gradient, pushing weights to zero more aggressively

> [!note]- Answer
> **B**
>
> - **A 错**：L2 把权重压小但不会压到精确的 0——球形约束域光滑，等高线在球面任意点相切，不存在产生精确零的"角点效应"。
> - **B 正确**：L1 约束域是菱形，顶点落在坐标轴上（某些权重 = 0）。Loss 等高线与菱形相切时极大概率切于顶点，产生稀疏解，实现自动特征选择。
> - **C 错**：L2 优化更稳定是真的，但稳定性和稀疏性是两回事；这里需求是特征选择，不是优化稳定性。
> - **D 错**：L1 产生稀疏解的原因是几何性质（菱形角点），不是梯度大小。

**Question 2.** *(BN 训练 vs 推理 ★★★)* A model with Batch Normalisation performs well during training but produces inconsistent outputs after deployment, where requests arrive one sample at a time. What is the most likely cause?

- [ ] A. Single-sample inference causes BN to estimate mean and variance from one data point, making the normalisation statistics unreliable
- [ ] B. The learned γ and β parameters are not updated during inference, so BN reverts to standardising with zero mean and unit variance
- [x] C. BN is intended to use training-time running statistics at inference; if the model remains in training mode, it computes per-batch statistics instead, which are unstable for single samples
- [ ] D. BN's numerical stability term ε accumulates across forward passes, introducing a systematic offset that grows with inference volume

> [!note]- Answer
> **C**
>
> - **A** 描述了现象（单样本统计不可靠），但它是 C 的结果，不是独立原因。根本原因是没有切换到 eval 模式。
> - **B 错**：γ 和 β 是训练好的可学习参数，推理时不会被重置为 0 和 1。
> - **C 正确**：推理时必须调用 `model.eval()` 让 BN 切换为使用 running mean/variance。若仍在 training 模式，单样本推理等于用 1 个样本估计统计量，结果极不稳定。
> - **D 错**：ε 是固定的数值稳定常数（如 1e-5），不会随推理次数累积。

**Question 3.** *(四种 Normalization 对比 ★★)* A team is building a language model using a Transformer. They find that Batch Normalisation causes unstable training, especially with small batch sizes. Which normalisation method is most suitable, and why?

- [ ] A. Instance Normalisation; it normalises each sample independently, removing the batch-size dependency entirely
- [x] B. Layer Normalisation; it normalises across all features within each sample, making it independent of batch size and stable for variable-length sequences
- [ ] C. Group Normalisation; it divides channels into groups and is the standard choice for sequence models when batch size is small
- [ ] D. Batch Normalisation with a larger batch size; the instability is a hyperparameter issue, not a fundamental incompatibility with Transformers

> [!note]- Answer
> **B**
>
> - **A 错**：Instance Norm 在 H、W 维度归一化（针对空间特征图），用于风格迁移等图像任务，不适合 Transformer 的序列特征。
> - **B 正确**：Layer Norm 对每个 token 的所有特征维度归一化，不依赖 batch size，且序列长度变化不影响统计计算，是 Transformer 的标准选择。
> - **C 错**：Group Norm 常用于 batch 小的图像任务（如目标检测），不是序列模型的标准选择。
> - **D 错**：BN 对 RNN/Transformer 存在根本性问题——序列长度不固定使 batch 统计不稳定，增大 batch size 也无法解决。

**Question 4.** *(Dropout + Ensemble ★★★)* A recommendation system engineer is evaluating two approaches to reduce overfitting: (A) train 10 independent models and average their predictions; (B) add Dropout ($p = 0.5$) to a single model. **(4 Marks)**

(a) How can Dropout be interpreted through the lens of ensemble learning? **(2 Marks)**

(b) Inverted Dropout divides surviving activations by $(1-p)$ during training. Why does this mean no rescaling is needed at inference time? **(2 Marks)**

> [!note]- Answer
> **(a)**
> 每次前向传播使用不同的随机 mask，等价于训练一个不同结构的子网络。对于 $n$ 个神经元，理论上存在 $2^n$ 种 mask——Dropout 隐式训练了所有这些子网络（共享权重）。推理时使用完整网络，近似于对所有子网络预测取**隐式 Ensemble 平均**，比方案 A 更高效（无需存储 10 个模型）。
>
> **(b)**
> Inverted Dropout 在**训练时**就对保留的激活值除以 $(1-p)$，使得每个激活的**期望值**等于无 Dropout 时的值（$E[\text{output}] = (1-p) \cdot \frac{x}{1-p} = x$）。推理时不 Dropout，网络输出的期望与训练时一致，无需额外缩放。

**Question 5.** *(Early Stopping + Data Augmentation ★★)* A model's training loss continues to fall while validation loss has been rising for 10 epochs. **(4 Marks)**

(a) What does this pattern indicate, and what should Early Stopping do at this point? **(2 Marks)**

(b) A colleague suggests Data Augmentation as an alternative. How does it address overfitting differently from Early Stopping? **(2 Marks)**

> [!note]- Answer
> **(a)**
> 训练 loss 下降、验证 loss 上升，说明模型开始**过拟合**——在拟合训练集的噪声，而不是学习可泛化的特征。Early Stopping 应在验证 loss 最低点停止训练，并恢复该时刻的参数。本质上是限制了训练的有效轮次，等价于约束了模型的有效容量（隐式正则化）。
>
> **(b)**
> Early Stopping 通过**限制训练时长**控制过拟合，不改变数据本身。Data Augmentation 通过**扩充训练数据**（翻转、裁剪、白化等变换）减少过拟合——让模型见到同一语义的更多变体，迫使它学习变换不变的本质特征，而不是记忆特定的表面形式。两者机制不同，可以同时使用。

**Question 6.** *(BN forward + backward ★★★)* Implement the forward and backward passes of a Batch Normalisation layer **without using autograd**. **(8 Marks)**

```python
class BatchNorm:
    def __init__(self, num_features):
        self.gamma = torch.ones(1, num_features)
        self.beta  = torch.zeros(1, num_features)
        self.eps   = 1e-5

    def forward(self, z):

        # Compute mu, var (unbiased=False), std, z_hat.
        # Apply gamma and beta. Cache what you need for backward.
        ...

    def backward(self, grad):
        # grad: (B, F) — upstream gradient
        # Compute dgamma, dbeta stored on self.
        # Return dz (gradient w.r.t. z).
        ...
```

**(4 Marks)** `forward`: write the complete computation of `mu`, `var`, `std`, `z_hat`, and the final output

**(4 Marks)** `backward`: write the complete formulas for `dgamma`, `dbeta`, and `dz`

> [!note]- Answer
> ```python
> def forward(self, z):
>     B = z.shape[0]
>     mu    = z.mean(dim=0, keepdim=True)
>     var   = z.var(dim=0, keepdim=True, unbiased=False)
>     std   = torch.sqrt(var + self.eps)
>     z_hat = (z - mu) / std
>     self._cache = (z_hat, std, B)
>     return self.gamma * z_hat + self.beta
>
> def backward(self, grad):
>     z_hat, std, B = self._cache
>     self.dgamma = (grad * z_hat).sum(dim=0, keepdim=True)
>     self.dbeta  =  grad.sum(dim=0, keepdim=True)
>     dz_hat = grad * self.gamma
>     dz = (1.0 / (B * std)) * (
>         B * dz_hat
>         - dz_hat.sum(dim=0, keepdim=True)
>         - z_hat * (dz_hat * z_hat).sum(dim=0, keepdim=True)
>     )
>     return dz
> ```
>
> **关键点**：
> - `var` 必须用 `unbiased=False`（除以 $B$，不是 $B-1$）
> - `dz` 的三项来自完整链式法则展开：$B \cdot dz\_hat$（直接项）$- \sum dz\_hat$（均值项）$- z\_hat \cdot \sum(dz\_hat \cdot z\_hat)$（方差项）
> - 忘记任意一项得 0 分（该项）

**Question 7.** *(L1 vs L2 论述 ★★★)* A data science team is choosing a regularisation strategy for two different projects:

- **Project A**: Gene expression prediction — 10,000 input features, but domain experts believe only ~50 genes are relevant
- **Project B**: House price prediction — 30 input features, most of which are known to have some influence, differing only in magnitude

Analyse which regularisation strategy is appropriate for each project from the following perspectives. **(8 Marks)**

(a) How does the geometry of the L1 vs L2 constraint region determine whether sparse solutions arise? **(3 Marks)**

(b) What prior distribution does each regulariser correspond to in the Bayesian interpretation, and what assumption does each make about the weights? **(2 Marks)**

(c) Give a concrete recommendation for Project A and Project B, with justification. **(3 Marks)**

> [!note]- Answer
> **(a) 几何分析（3分）**
>
> L1 约束域是**菱形**（高维超八面体），顶点落在坐标轴上（即某些权重精确为 0）。Loss 等高线与菱形相切时，极大概率切于顶点，产生**稀疏解**（多数权重 = 0）。
>
> L2 约束域是**光滑球**，等高线可在球面任意位置相切，权重被均匀缩小趋近于 0，但几乎不会精确等于 0，**不产生稀疏解**。
>
> **(b) 贝叶斯先验（2分）**
>
> | 正则化 | 对应先验 | 假设 |
> |--------|---------|------|
> | L2 | 高斯分布 $P(w) \propto e^{-\frac{\lambda}{2}w^2}$ | 权重集中于 0 附近，大权重不太可能但不排除 |
> | L1 | Laplace 分布 $P(w) \propto e^{-\lambda\|w\|}$ | 大多数权重接近 0，少数权重可以较大，天然鼓励稀疏 |
>
> **(c) 建议（3分）**
>
> **Project A → L1**：输入特征极多（10,000）但真正相关的极少（~50），本质是稀疏特征选择问题。L1 能将无关特征权重压至精确 0，等价于自动特征选择，与领域专家的先验知识吻合。
>
> **Project B → L2**：30 个特征大部分都有贡献，不需要稀疏，只需防止权重过大导致过拟合。L2 均匀缩小所有权重、保留全部特征、优化稳定（可微），是"大部分特征有效"场景的标准选择。
