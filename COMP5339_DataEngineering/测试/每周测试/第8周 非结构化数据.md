**Question 1.** A news aggregator platform indexes millions of articles daily. The search team is evaluating retrieval methods for two planned features:
- **Feature A**: "Find all articles containing the exact keyword 'climate'"
- **Feature B**: "Find articles semantically similar to 'rising sea levels'" — should match articles about coastal flooding even if they don't share the same words

Which statement correctly describes the best approach for each feature?

- [ ] A. Use TF-IDF for both — it handles keyword and semantic search equally well
- [ ] B. Use Inverted Index + TF-IDF for Feature A; use Vector Embeddings (e.g., BERT) + cosine similarity for Feature B
- [ ] C. Use BERT for both — it is always more accurate than TF-IDF
- [ ] D. Use TF-IDF for Feature B and BERT for Feature A, since TF-IDF better captures semantic meaning

> [!note]- Answer
> **B. Inverted Index / TF-IDF for keyword search; Vector Embeddings for semantic search。**
>
> - **TF-IDF + Inverted Index**：基于词频统计，是关键词精确匹配的标准方案。Feature A 需要精确定位含 "climate" 的文章，Inverted Index 直接查词表，高效且准确。
> - **Vector Embeddings (BERT)**：将文本映射为稠密向量，语义相近的文本在向量空间中相邻。Feature B 需要理解"rising sea levels"与"coastal flooding"的语义关联——TF-IDF 是词袋模型（Bag of Words），两段文字共享词少则相似度接近 0；BERT 的 contextual embeddings 能捕捉语义。
>
> BERT 并非在所有任务上都优于 TF-IDF——关键词精确搜索用 Inverted Index 更快，无需 GPU 推理。

**Question 2.** A medical imaging startup stores grayscale chest X-rays (512×512 pixels). The team is choosing a feature extraction approach before training a diagnostic classifier:
- **Approach A**: Pre-trained CNN (ResNet) → 2048-dimensional dense feature vector per image
- **Approach B**: Handcrafted features — pixel intensity histograms + Canny edge detection outputs

A junior engineer argues Approach B is more trustworthy because *"every number in the vector has a clear meaning."* The team has a large labelled dataset and GPU compute available. Which statement is most accurate?

- [ ] A. Approach B is always preferred in medical settings because interpretability is legally required
- [ ] B. Approach A and B will achieve similar accuracy since both represent the same image content
- [x] C. Approach A (Black box CNN) will likely achieve higher accuracy given sufficient labelled data and GPU; Approach B (White box) is more interpretable but has limited expressive power
- [ ] D. Approach B is strictly better for production — handcrafted features are faster at inference time

> [!note]- Answer
> **C. Black box（CNN）vs White box（手工特征）的核心权衡。**
>
> - **White box（Approach B）**：手工设计特征可解释，但表达能力有限，依赖专家设计，难以捕捉复杂的病理模式。
> - **Black box（Approach A）**：CNN 自动学习层次化特征，不可解释，但在大量标注数据下性能大幅超越手工特征，是医学影像分类的主流方案。
>
> 注意原始维度：512×512 灰度图像展平后是 **262,144 维**——这正是为什么需要 CNN 做特征压缩（降至 2048 维）的原因。有大量数据和 GPU 时，Black box CNN 是正确选择。

**Question 3.** A journalist works with a dataset of 80,000 photos scraped from a public archive. She needs to: (1) filter photos taken in Sydney between 2019–2023; (2) identify photos that contain sensitive GPS coordinates that must be removed before publication. Which tool is most appropriate?

- [ ] A. A pre-trained CNN to classify image scene content and infer shooting location from visual context
- [ ] B. OpenCV to analyse pixel patterns and detect location from image composition
- [x] C. `exiftool` to extract the EXIF metadata embedded in each image file (GPS coordinates, capture timestamp, camera model)
- [ ] D. A TF-IDF index on image filenames to find date-related patterns

> [!note]- Answer
> **C. `exiftool` — EXIF Metadata Extraction（元数据提取）。**
>
> **EXIF（Exchangeable Image File Format）** 是嵌入在 JPEG / TIFF 文件中的结构化元数据，包含：
> - 拍摄时间（`DateTimeOriginal`）
> - GPS 坐标（`GPSLatitude` / `GPSLongitude`）
> - 相机品牌型号、快门速度、ISO 等
>
> `exiftool` 直接读取这些字段，无需分析像素内容——远比用 CNN 推断位置更精确、更高效。社交媒体上传前去除 GPS 信息也是 `exiftool` 的典型应用场景（隐私保护）。

**Question 4.** A tokenizer processes the sentence: *"GPT-4's performance is state-of-the-art!"* A developer using a **subword tokenizer (BPE / WordPiece)** notices the output splits "GPT-4" into multiple tokens. A colleague argues this is a bug — "GPT-4 should never be split." What does this behaviour actually demonstrate?

- [ ] A. BPE tokenizers always split all words into individual characters regardless of vocabulary
- [ ] B. The tokenizer made an error — named entities should always be preserved as single tokens
- [ ] C. "GPT-4" is partially out-of-vocabulary; subword tokenization breaks unknown words into known subword units, avoiding the OOV problem while preserving partial meaning
- [ ] D. Subword tokenization only applies to Chinese and Japanese; English words should never be split

> [!note]- Answer
> **C. Subword Tokenization（子词分词）解决 OOV 问题。**
>
> 词级分词的问题：词汇表（vocabulary）固定，训练集未见过的词只能替换为 `[UNK]`，丢失所有信息。
>
> **BPE / WordPiece** 的解决思路：将词分割为高频子词单元，使模型即使面对未见过的词（如新产品名称），也能通过组合已知子词来理解部分含义。`##` 前缀表示该 token 是词的非首部分（non-word-initial）。BERT 使用 WordPiece，GPT 系列使用 BPE。
>
> **Tokenization 是 NLP 管道的第一步**：`原始文本 → Tokens → Token IDs → Embeddings → Model`。

**Scenario 1:** A search engine startup indexes three research papers:
- Doc1: "data pipeline engineering"
- Doc2: "data science and machine learning"
- Doc3: "pipeline automation tools"

**Question 5:** **(6 Marks)**

**(a)** Calculate the TF-IDF score for the term **"pipeline"** in **Doc1**. Use natural log (ln). Show all steps. **(3 Marks)**

**(b)** Describe how an **Inverted Index** would be structured for these three documents. Show the index entry for the term "data", and explain how it enables faster search than full document scanning. **(3 Marks)**

> [!note]- Answer
> **(a) TF-IDF for "pipeline" in Doc1**
>
> **Step 1: TF（Term Frequency）**
> - Doc1 共 3 个词，"pipeline" 出现 1 次
> - TF("pipeline", Doc1) = 1/3 ≈ **0.333**
>
> **Step 2: IDF（Inverse Document Frequency）**
> - 总文档数 N = 3
> - 含 "pipeline" 的文档：Doc1 和 Doc3，DF = 2
> - IDF("pipeline") = ln(3/2) = ln(1.5) ≈ **0.405**
>
> **Step 3: TF-IDF**
> - TF-IDF = TF × IDF = 0.333 × 0.405 ≈ **0.135**
>
> 解读："pipeline" 是 Doc1 的重要词，但因同时出现在 Doc3，IDF 适中。若 "data" 出现在全部3篇，IDF = ln(3/3) = 0 → TF-IDF = 0（停用词效果）。
>
> **(b) Inverted Index 结构**
>
> ```
> "data"      → [(Doc1, tf=1, pos:0), (Doc2, tf=1, pos:0)]
> "pipeline"  → [(Doc1, tf=1, pos:1), (Doc3, tf=1, pos:0)]
> "science"   → [(Doc2, tf=1, pos:1)]
> "automation"→ [(Doc3, tf=1, pos:1)]
> ```
>
> "data" 的索引条目：`"data" → [Doc1, Doc2]`
>
> **搜索加速原理**：查询 "data pipeline" 时：
> 1. 查 Index 找 "data" → {Doc1, Doc2}
> 2. 查 Index 找 "pipeline" → {Doc1, Doc3}
> 3. 求交集 → {Doc1}
>
> 无需扫描任何文档内容，时间复杂度从 O(N×文档长度) 降为 O(1) 索引查找 + O(k) 集合运算。Elasticsearch、PostgreSQL `tsvector` 均基于此原理。

**Scenario 2:** A customer service team at a retailer wants to build two NLP features:
- **Feature X**: Automatically tag support tickets with labels like "refund", "shipping", "login error"
- **Feature Y**: Match customer questions to the most semantically similar FAQ entry — e.g., *"I can't get into my account"* should match *"How do I reset my password?"*

**Question 6:** **(4 Marks)**

**(a)** Which text representation method — TF-IDF or BERT dense embeddings — is more suitable for each feature, and why? **(2 Marks)**

**(b)** TF-IDF produces a **sparse vector** while BERT produces a **dense vector**. Explain the difference and why it matters for storage and similarity search at scale. **(2 Marks)**

> [!note]- Answer
> **(a) Feature 对应方案**
>
> - **Feature X（关键词标注）→ TF-IDF**：标注是词汇精确匹配任务——某工单含高权重词 "refund" 则标为退款类。TF-IDF 轻量、可解释、无需 GPU，足以完成此任务。
> - **Feature Y（语义相似匹配）→ BERT Embeddings**：用户措辞与 FAQ 标题可能完全不重叠（"can't get into my account" vs "reset password"），TF-IDF 共享词极少，相似度趋近 0。BERT 的 contextual embeddings 将语义相近文本映射到向量空间相近位置，余弦相似度能捕捉语义关联。
>
> **(b) Sparse vs Dense 向量的差异**
>
> - **TF-IDF → Sparse Vector（稀疏向量）**：维度 = 词汇表大小（可达数万维），每个文档中绝大多数词 TF-IDF = 0，向量大部分为零。可用稀疏格式（只存非零项）节省存储，但高维稀疏导致无法捕捉语义。
>
> - **BERT → Dense Vector（稠密向量）**：固定低维度（如 768 维），每维均非零，携带语义信息。存储固定（768 × float32 ≈ 3KB/向量），大规模语料库（百万文档）需要向量数据库（如 Faiss、pgvector）做 ANN（Approximate Nearest Neighbour）搜索来实现高效相似度查询。

**Question 7.** An NLP engineer is selecting a pre-trained model for two tasks: (1) named entity recognition (NER) on medical records; (2) generating discharge summary drafts from structured notes. Which assignment is correct?

- [ ] A. BERT for both — it is pre-trained on more data than GPT
- [ ] B. GPT for NER; BERT for generation
- [x] C. BERT for NER; GPT for generation
- [ ] D. Either works equally well for both tasks

> [!note]- Answer
> **C. BERT for NER; GPT for generation.**
>
> | | BERT | GPT |
> |--|------|-----|
> | **架构** | Bidirectional encoder-only | Unidirectional decoder |
> | **训练目标** | Masked Language Model（预测遮盖词） | Next Word Prediction |
> | **擅长** | 理解类：Classification、NER、Sentiment、QA | 生成类：Chat、Summarization、Drafting |
> | **适配方式** | Fine-tuning（有标注数据） | Few-shot / Zero-shot |
>
> NER 是序列标注任务，需要模型理解每个词在完整句子上下文中的含义——BERT 双向注意力天然适合。摘要生成是生成任务，需要 GPT 的左到右解码能力。A 错：参数量多少不决定任务适配性。B 颠倒了两者的定位。D 错：架构的根本差异导致任务表现截然不同。
