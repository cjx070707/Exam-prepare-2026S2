## 参考答案

1. **B**
2. **B**
3. **C**
4. **B**
5. **B**
6.
   - **Schema-on-write**：写入前必须定义好表结构，数据必须符合 schema 才能存入（如 Data Warehouse）
   - **Schema-on-read**：数据以原始格式存储，读取时才应用 schema 解析（如 Data Lake）
7. 示例：
   - `Sales(sale_id, time_id, product_id, market_id, sales_amt)`（Fact）
   - `Time(time_id, date, week, month, quarter, year)`
   - `Product(product_id, name, category, price)`
   - `Market(market_id, city, state, region)`
8. Data Swamp 是 Data Lake 退化后的状态：数据无组织堆积、无元数据、无治理，导致数据无法被找到或使用。原因：缺乏数据目录、无质量控制、无访问权限管理。避免：建立数据目录（Data Catalog）、强制元数据标注、定期数据质量检查。
9. 问题：SQL 注入漏洞，字符串拼接不安全。修改：
   ```python
   cursor.execute("SELECT * FROM Sales WHERE region = %s", (region,))
   ```
10. 实时订单系统 → **OLTP**：高频点查询和写入，需要低延迟和 ACID 事务。月度分析报表 → **OLAP**：扫描海量历史数据做聚合，读多写少，对延迟容忍度高。
