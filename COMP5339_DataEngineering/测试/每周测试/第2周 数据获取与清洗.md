## 选择题

**1.** 一家公司需要每隔 15 分钟从外部天气 API 拉取最新气象数据，写入内部数据仓库。这种数据摄取模式最准确的描述是？

- A. Push — 外部 API 主动将数据推送到公司系统
- B. Pull — 公司系统向 API 发出请求并接收响应，无固定时间间隔
- C. Poll — 公司系统按固定时间间隔周期性地向 API 发起请求
- D. CDC — 基于数据库变更日志捕获增量数据

> [!tip]- 答案
> **答案：C**
> 解析：Poll（轮询）是 Pull 的周期性变体——消费方按固定时间间隔主动发起请求。Pull 泛指消费方主动拉取，不强调周期性；Push 是生产方主动推送；CDC 监听数据库日志，与 API 拉取无关。"每隔 15 分钟"是 Poll 的典型特征。

**2.** Apache Kafka 作为消息队列，其数据摄取模式属于以下哪一类？

- A. Poll，因为消费者需要定期检查队列中是否有新消息
- B. Push，因为 Kafka broker 主动将消息推送给订阅的消费者
- C. Pull，因为消费者可以按自己的节奏从 partition 中读取消息
- D. CDC，因为 Kafka Connect 可以连接数据库

> [!tip]- 答案
> **答案：B**
> 解析：Kafka 被归类为 Push 模式——生产者将消息推送到 broker，broker 再推送给订阅的消费者组。虽然消费者也可以"拉取"的方式读取，但 Kafka 的设计哲学和课程定义将其归为 Push/Stream 推送模型。注意 D 是混淆项：Kafka Connect 可以与数据库集成，但这不影响 Kafka 本身的摄取模式分类。

**3.** 以下哪个说法正确描述了 Bounded Data 和 Unbounded Data 的区别？

- A. Bounded Data 只能存储在关系型数据库中；Unbounded Data 必须存储在 NoSQL 中
- B. Bounded Data 是有明确起止点的有限数据集；Unbounded Data 是持续不断产生、无固定终点的数据流
- C. Unbounded Data 总是比 Bounded Data 数据量更大
- D. Bounded Data 不能用于机器学习训练；Unbounded Data 才能用于实时预测

> [!tip]- 答案
> **答案：B**
> 解析：Bounded（有界）数据有明确的起止边界，如历史交易记录、一次数据库快照；Unbounded（无界）数据持续产生，如传感器数据流、社交媒体实时信息流。大小不是判断标准（C 错）；存储方式也不是定义特征（A 错）。

**4.** 关于 Unbounded Data 的处理，以下哪个表述是正确的？

- A. Unbounded Data 无法被处理，只能实时丢弃超出内存的部分
- B. 处理 Unbounded Data 前，必须先将其转换为 Bounded Data（如通过窗口切片）
- C. Unbounded Data 只能用 Apache Flink 处理，其他框架不支持
- D. Unbounded Data 和 Bounded Data 的处理方式完全相同，没有区别

> [!tip]- 答案
> **答案：B**
> 解析：课程核心概念：**所有 Unbounded 数据在处理前必须先转换为 Bounded 数据**，最常见的方式是"窗口化"（windowing）——将无限流切分为有限的时间窗口（如每5分钟一个窗口）再处理。A/C/D 均不正确。

**5.** 数据工程中将数据来源分为五类。以下哪个选项将"来源类型"与"典型例子"正确匹配？

- A. CDC + Logs — 通过 REST API 调用第三方支付平台获取交易数据
- B. Messages + Streams — 数据库的 binlog 记录每次 INSERT/UPDATE/DELETE 操作
- C. File Access — 从 FTP 服务器下载每日生成的 CSV 销售报表
- D. Database — 使用 Scrapy 爬取电商网站的商品价格

> [!tip]- 答案
> **答案：C**
> 解析：File Access 指通过文件系统或文件传输协议（FTP/SFTP/S3）获取结构化文件，CSV 报表是典型例子。A 描述的是 API 来源；B 描述的是 CDC（Change Data Capture）；D 描述的是 Web Scraping，属于 API 或特殊 File Access，不属于 Database 来源。

**6.** 在数据质量评估框架中，以下哪一项属于 **Inconsistent（不一致）** 问题，而非其他类型的数据质量问题？

- A. 客户年龄字段中存在大量 NaN 空值
- B. 用户性别字段中，"Male" 和 "M" 在不同数据源中指代相同含义
- C. 产品价格字段中出现了 -500 元的负值
- D. 日期字段中有一条记录显示为 "2099-01-01"，远超合理范围

> [!tip]- 答案
> **答案：B**
> 解析：Inconsistent 问题指同一实体在不同数据源或记录中用不同格式/编码表示，"Male" vs "M" 是典型的编码不一致。A 是 Missing 问题；C 和 D 是 Incorrect 问题（超出合理值域）。Inconsistent 的 remedy 是 **reconcile**（协调统一），而非简单删除。

**7.** 以下哪种图表类型最适合展示"2024年各月份网站访问量的分布情况"（月份为离散类别）？

- A. Scatter Plot（散点图）——因为数据点彼此独立
- B. Histogram（直方图）——因为访问量是连续数值
- C. Bar Chart（柱状图）——因为月份是离散的类别变量
- D. Line Chart（折线图）——因为数据有时间先后顺序

> [!tip]- 答案
> **答案：C**
> 解析：Bar Chart 用于展示**离散类别（nominal/ordinal）**之间的数量比较，月份是离散类别，柱状图最合适。Histogram 用于连续数值的分布（如访问时长的频率分布）；Scatter Plot 展示两个连续变量的关系；Line Chart 虽然也可展示时间趋势，但当重点是"各月份数量比较"而非"趋势"时，Bar Chart 更准确。

**8.** Benford's Law 在数据质量管理中的主要用途是？

- A. 检测数据集中的缺失值，判断缺失是否为随机发生
- B. 识别数值数据中可能存在人为篡改或欺诈的 Incorrect 数据
- C. 将不同来源的数据格式统一化，解决 Inconsistent 问题
- D. 确定最优的缺失值填充策略（均值 vs. 中位数）

> [!tip]- 答案
> **答案：B**
> 解析：Benford's Law 描述了自然产生的数值数据中首位数字的分布规律（1 出现概率约 30%，9 约 4.6%）。若数据集中的首位数字分布严重偏离该规律，提示数据可能被人为修改或伪造，常用于**欺诈检测**。它不用于检测缺失值或解决格式不一致问题。

## 简答题

**9.** 请解释数据摄取的三种模式（Push、Pull、Poll）的定义，并说明 Web Crawling 属于哪种模式及理由。

> [!tip]- 答案
> - **Push**：数据生产方主动将数据推送到消费方，消费方被动接收。例：Kafka broker 将消息推送给订阅者。
> - **Pull**：数据消费方主动向生产方发起请求并拉取数据，不强调时间规律。例：按需调用 REST API。
> - **Poll（轮询）**：Pull 的周期性变体，消费方按**固定时间间隔**定期发起请求。例：每小时抓取一次股价数据。
>
> **Web Crawling 属于 Poll 模式。** 爬虫通常按预设时间间隔（如每天、每周）重复抓取目标网站，是消费方主动、周期性发起请求的行为，符合 Poll 的定义。

**10.** 数据质量框架包含六个维度。请列出至少五个维度，并针对"缺失值（Missing Data）非随机发生"这一场景，说明它会影响哪个维度，以及对下游分析的具体危害。

> [!tip]- 答案
> **六个数据质量维度**（至少列出五个）：
> 1. **Believable（可信性）** — 数据是否来自可靠来源、用户是否信任它
> 2. **Value-Added（增值性）** — 数据是否对业务决策有实际价值
> 3. **Relevant（相关性）** — 数据是否与分析目标相关
> 4. **Accurate（准确性）** — 数据是否正确反映现实
> 5. **Interpretable（可解释性）** — 数据是否有足够的文档/元数据供理解
> 6. **Accessible（可访问性）** — 数据是否易于获取和使用（第6个维度）
>
> **非随机缺失的危害：**
> 缺失值若**非随机（non-random）**发生，主要损害 **Accurate** 和 **Believable** 维度，并导致**代表性偏差（underrepresentation bias）**。例如：若问卷中收入越高的人越倾向于不填写收入字段，则删除这些缺失值后，样本中高收入群体严重不足，任何基于该数据的统计分析或模型都会系统性地低估高收入群体的行为特征，结论不可信。

**11.** 以下是一段 HTML 片段，请说明其结构，并解释数据工程师在爬取该页面时，如何定位所有产品的名称和价格。

```html
<div class="product-list">
  <div class="product-item">
    <span class="product-name">无线耳机 Pro</span>
    <span class="product-price">¥299</span>
  </div>
  <div class="product-item">
    <span class="product-name">便携充电宝</span>
    <span class="product-price">¥89</span>
  </div>
</div>
```

> [!tip]- 答案
> **HTML 结构解析：**
> - 最外层 `<div class="product-list">` 是产品列表容器。
> - 内部每个 `<div class="product-item">` 代表一个产品条目。
> - 每个条目内有两个 `<span>`：`class="product-name"` 存储产品名称，`class="product-price"` 存储价格。
>
> **定位方式：**
> - **CSS Selector**：使用 `.product-item .product-name` 选取所有产品名称；`.product-item .product-price` 选取所有价格。
> - **XPath**：`//div[@class="product-item"]/span[@class="product-name"]` 选取所有名称节点，提取其 `text()` 内容。
> - 实际爬取时，先找到所有 `.product-item` 节点，再对每个节点分别提取子元素的文本，形成结构化的（名称, 价格）对。
