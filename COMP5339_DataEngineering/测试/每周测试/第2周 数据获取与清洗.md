## 选择题

**1.** 数据质量问题中，"Default 问题"指的是？

- A. 数据字段完全为空（NULL）
- B. 用占位符（如 0、99999、"N/A"）代替真正的缺失值
- C. 数据格式不统一（如日期格式混用）
- D. 同一记录在数据库中存储了多次

> [!tip]- 答案
>
> **答案：B**
>
> 解析：Default 问题是指本应为缺失的值被一个"默认占位符"填充，如用 `99999` 表示未知年龄、用 `"N/A"` 表示未知城市。这类值表面上非空，但实际无效，清洗时需先识别并替换为真正的 NaN。
>

**2.** 以下哪种数据清洗操作属于 "Normalization"（标准化）？

- A. 删除年龄列中值为 NaN 的行
- B. 将 "Sydney"、"sydney"、"SYDNEY" 统一为 "Sydney"
- C. 用均值填充缺失的身高数据
- D. 检测并删除重复的用户记录

> [!tip]- 答案
>
> **答案：B**
>
> 解析：Normalization 是将数据转换为一致的标准格式，消除因大小写、缩写、拼写不一致导致的问题。A 是删除缺失值；C 是 Imputation（填充）；D 是 Deduplication。
>

**3.** 以下 Pandas 操作中，哪个用于删除含缺失值的行？

- A. `df.fillna(0)`
- B. `df.replace(0, None)`
- C. `df.dropna()`
- D. `df.astype(int)`

> [!tip]- 答案
>
> **答案：C**
>
> 解析：`dropna()` 删除含 NaN 的行（默认）或列；`fillna(0)` 用0填充缺失值；`replace` 替换特定值；`astype` 做类型转换。
>

**4.** 为什么 Pandas 中不能直接将含 NaN 的列转为 `int` 类型？

- A. int 类型不支持负数
- B. NaN 是 float 类型，int 无法表示缺失值
- C. Pandas 不支持整数列
- D. astype 只能用于 str 类型

> [!tip]- 答案
>
> **答案：B**
>
> 解析：Python/NumPy 的 `int` 类型没有 NaN 值的概念，NaN 本身是 float。含 NaN 的列默认为 float64，直接 `astype(int)` 会报错。解决方案：先 `fillna()` 再转型，或使用 Pandas 的 nullable integer 类型 `pd.Int64Dtype()`。
>

**5.** `df['city'].replace('N/A', 'Unknown')` 和 `df['city'].fillna('Unknown')` 的区别是？

- A. 两者完全等价，可以互换使用
- B. `replace` 替换特定字符串值；`fillna` 只填充 Python 的 NaN 空值
- C. `fillna` 替换所有值；`replace` 只替换第一个匹配
- D. `replace` 只能用于数值列

> [!tip]- 答案
>
> **答案：B**
>
> 解析：`fillna('Unknown')` 只替换真正的 NaN（Python None/np.nan）；`replace('N/A', 'Unknown')` 替换字符串 `'N/A'`。若缺失值以字符串 `'N/A'` 形式存储（非 NaN），用 `fillna` 无效，必须用 `replace`。
>

**6.** 以下哪种场景最适合使用 Mean Imputation（均值填充）？

- A. 类别列（如城市名称）存在缺失
- B. 数值列的缺失比例很高（>50%）且数据分布高度偏斜
- C. 数值列的缺失比例较低且数据分布近似正态
- D. 所有缺失值都应该用均值填充

> [!tip]- 答案
>
> **答案：C**
>
> 解析：Mean Imputation 适合缺失比例低、分布对称的数值列——此时均值接近真实值。若分布偏斜，均值不代表典型值，应用中位数（Median Imputation）；若缺失比例高，填充后引入偏差过大；类别列不能用均值。
>

**7.** 正则表达式 `^\d{4}-\d{2}-\d{2}$` 能匹配以下哪个字符串？

- A. `2024/01/15`
- B. `2024-1-5`
- C. `2024-01-15`
- D. `24-01-15`

> [!tip]- 答案
>
> **答案：C**
>
> 解析：`\d{4}` 匹配4位数字，`-` 匹配连字符，`\d{2}` 匹配2位数字，`^` 和 `$` 锚定行首行尾。只有 `2024-01-15` 满足 4位-2位-2位格式。`2024-1-5` 月份和日期不足2位，不匹配。
>

**8.** 以下代码的问题是什么？

```python
df['age'] = df['age'].astype(int)
```

假设 `age` 列中存在 NaN 值。

- A. 没有问题，astype 会自动将 NaN 转为 0
- B. 会报错，因为 NaN 无法转换为 int
- C. 会将 NaN 转为 -1
- D. 只有 Python 2 中才会报错

> [!tip]- 答案
>
> **答案：B**
>
> 解析：`astype(int)` 遇到 NaN 会抛出 `ValueError`。正确做法是先填充再转型：
> ```python
> df['age'] = df['age'].fillna(0).astype(int)
> ```
> 或使用 nullable int：
> ```python
> df['age'] = df['age'].astype(pd.Int64Dtype())
> ```
>

**9.** 列出数据质量的至少四个维度，并分别说明含义及一个对应问题的例子。

> [!tip]- 答案
>
> | 维度 | 含义 | 问题示例 |
> |------|------|----------|
> | **Accuracy（准确性）** | 数据是否反映真实情况 | 用户年龄录入为 999 |
> | **Completeness（完整性）** | 是否缺少必要字段 | 订单表中运费字段大量为空 |
> | **Consistency（一致性）** | 同一数据在不同来源是否一致 | 两个系统中同一客户的姓名格式不同 |
> | **Timeliness（时效性）** | 数据是否足够新 | 使用两年前的客户地址发快递 |
> | **Uniqueness（唯一性）** | 是否存在重复记录 | 同一用户注册了两个账号 |

**10.** 你有如下 DataFrame：

| name | age | city |
|------|-----|------|
| Alice | 25 | Sydney |
| Bob | 99999 | None |
| Carol | NaN | Melbourne |

写出 Pandas 代码，完成：① 将 `age` 列中值为 99999 的替换为 NaN；② 用 0 填充 `age` 的缺失值并转为 int；③ 用 "Unknown" 填充 `city` 的缺失值。

> [!tip]- 答案
>
> ```python
> import numpy as np
>
> # ① 将 99999 替换为 NaN
> df['age'] = df['age'].replace(99999, np.nan)
>
> # ② 填充缺失值并转为 int
> df['age'] = df['age'].fillna(0).astype(int)
>
> # ③ 填充城市缺失值
> df['city'] = df['city'].fillna('Unknown')
> ```
>
> 注意：`None`（Python 对象空值）和 `NaN`（float 类型）在 Pandas 中都被 `fillna` 识别为缺失。
>

**11.** 什么是 Deduplication（去重）？在数据清洗中，完全重复和近似重复（fuzzy duplicate）有什么区别，各如何处理？

> [!tip]- 答案
>
> **Deduplication** 是识别并删除数据集中重复记录的过程，确保每个实体只出现一次。
>
> - **完全重复（Exact duplicate）**：所有字段完全相同。处理方法：`df.drop_duplicates()`。
> - **近似重复（Fuzzy duplicate）**：记录指同一实体但存在拼写/格式差异（如 "John Smith" vs "J. Smith"）。处理方法：使用字符串相似度算法（如编辑距离 Levenshtein distance）或专门的 record linkage 库进行匹配合并。
>
> 近似重复更难自动检测，通常需要领域知识和额外的匹配规则。
>
