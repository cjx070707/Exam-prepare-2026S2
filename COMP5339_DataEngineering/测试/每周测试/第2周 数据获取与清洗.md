## Multiple Choice

**1.** A company needs to pull the latest weather data from an external weather API every 15 minutes and write it into an internal data warehouse. Which term most accurately describes this data ingestion pattern?

- A. Push — the external API actively pushes data to the company's system
- B. Pull — the company's system sends a request to the API and receives a response, with no fixed time interval
- C. Poll — the company's system periodically sends requests to the API at a fixed time interval
- D. CDC — incremental data is captured based on database change logs

> [!tip]- Answer
> **答案：C**
> Poll（轮询）是 Pull 的周期性变体——消费方按固定时间间隔主动发起请求。Pull 泛指消费方主动拉取，不强调周期性；Push 是生产方主动推送；CDC 监听数据库日志，与 API 拉取无关。"每隔 15 分钟"是 Poll 的典型特征。

**2.** Apache Kafka, used as a message queue, belongs to which data ingestion pattern?

- A. Poll, because consumers need to periodically check the queue for new messages
- B. Push, because the Kafka broker actively pushes messages to subscribed consumers
- C. Pull, because consumers can read messages from a partition at their own pace
- D. CDC, because Kafka Connect can connect to databases

> [!tip]- Answer
> **答案：B**
> Kafka 被归类为 Push 模式——生产者将消息推送到 broker，broker 再推送给订阅的消费者组。虽然消费者也可以"拉取"的方式读取，但 Kafka 的设计哲学和课程定义将其归为 Push/Stream 推送模型。注意 D 是混淆项：Kafka Connect 可以与数据库集成，但这不影响 Kafka 本身的摄取模式分类。

**3.** Which statement correctly describes the difference between Bounded Data and Unbounded Data?

- A. Bounded Data can only be stored in relational databases; Unbounded Data must be stored in NoSQL
- B. Bounded Data is a finite dataset with a clear start and end point; Unbounded Data is a continuously generated stream with no fixed endpoint
- C. Unbounded Data always has a larger volume than Bounded Data
- D. Bounded Data cannot be used for machine learning training; only Unbounded Data can be used for real-time prediction

> [!tip]- Answer
> **答案：B**
> Bounded（有界）数据有明确的起止边界，如历史交易记录、一次数据库快照；Unbounded（无界）数据持续产生，如传感器数据流、社交媒体实时信息流。大小不是判断标准（C 错）；存储方式也不是定义特征（A 错）。

**4.** Regarding the processing of Unbounded Data, which of the following statements is correct?

- A. Unbounded Data cannot be processed; data exceeding memory capacity must simply be discarded in real time
- B. Before processing Unbounded Data, it must first be converted into Bounded Data (e.g., via windowing)
- C. Unbounded Data can only be processed by Apache Flink; other frameworks do not support it
- D. Unbounded Data and Bounded Data are processed in exactly the same way, with no difference

> [!tip]- Answer
> **答案：B**
> 课程核心概念：**所有 Unbounded 数据在处理前必须先转换为 Bounded 数据**，最常见的方式是"窗口化"（windowing）——将无限流切分为有限的时间窗口（如每5分钟一个窗口）再处理。A/C/D 均不正确。

**5.** Data engineering classifies data sources into five categories. Which option correctly matches a "source type" with a "typical example"?

- A. CDC + Logs — obtaining transaction data from a third-party payment platform via a REST API call
- B. Messages + Streams — a database's binlog recording every INSERT/UPDATE/DELETE operation
- C. File Access — downloading daily-generated CSV sales reports from an FTP server
- D. Database — using Scrapy to scrape product prices from an e-commerce website

> [!tip]- Answer
> **答案：C**
> File Access 指通过文件系统或文件传输协议（FTP/SFTP/S3）获取结构化文件，CSV 报表是典型例子。A 描述的是 API 来源；B 描述的是 CDC（Change Data Capture）；D 描述的是 Web Scraping，属于 API 或特殊 File Access，不属于 Database 来源。

**6.** In a data quality assessment framework, which of the following is an example of an **Inconsistent** problem, rather than another type of data quality issue?

- A. A large number of NaN null values exist in the customer age field
- B. In the user gender field, "Male" and "M" refer to the same meaning across different data sources
- C. A value of -500 appears in the product price field
- D. A date field contains a record showing "2099-01-01", far outside a reasonable range

> [!tip]- Answer
> **答案：B**
> Inconsistent 问题指同一实体在不同数据源或记录中用不同格式/编码表示，"Male" vs "M" 是典型的编码不一致。A 是 Missing 问题；C 和 D 是 Incorrect 问题（超出合理值域）。Inconsistent 的 remedy 是 **reconcile**（协调统一），而非简单删除。

**7.** Which chart type is most appropriate for displaying "the distribution of monthly website visits throughout 2024" (where months are discrete categories)?

- A. Scatter Plot — because data points are independent of each other
- B. Histogram — because visit counts are continuous numerical values
- C. Bar Chart — because months are discrete categorical variables
- D. Line Chart — because the data has a chronological order

> [!tip]- Answer
> **答案：C**
> Bar Chart 用于展示**离散类别（nominal/ordinal）**之间的数量比较，月份是离散类别，柱状图最合适。Histogram 用于连续数值的分布（如访问时长的频率分布）；Scatter Plot 展示两个连续变量的关系；Line Chart 虽然也可展示时间趋势，但当重点是"各月份数量比较"而非"趋势"时，Bar Chart 更准确。

**8.** What is the primary use of Benford's Law in data quality management?

- A. Detecting missing values in a dataset to determine whether missingness is random
- B. Identifying potentially human-tampered or fraudulent Incorrect data in numerical datasets
- C. Standardizing data formats from different sources to resolve Inconsistent problems
- D. Determining the optimal missing value imputation strategy (mean vs. median)

> [!tip]- Answer
> **答案：B**
> Benford's Law 描述了自然产生的数值数据中首位数字的分布规律（1 出现概率约 30%，9 约 4.6%）。若数据集中的首位数字分布严重偏离该规律，提示数据可能被人为修改或伪造，常用于**欺诈检测**。它不用于检测缺失值或解决格式不一致问题。

## Short Answer

**9.** Explain the definitions of the three data ingestion patterns (Push, Pull, Poll), and state which pattern Web Crawling belongs to and why.

> [!tip]- Answer
> - **Push**：数据生产方主动将数据推送到消费方，消费方被动接收。例：Kafka broker 将消息推送给订阅者。
> - **Pull**：数据消费方主动向生产方发起请求并拉取数据，不强调时间规律。例：按需调用 REST API。
> - **Poll（轮询）**：Pull 的周期性变体，消费方按**固定时间间隔**定期发起请求。例：每小时抓取一次股价数据。
>
> **Web Crawling 属于 Poll 模式。** 爬虫通常按预设时间间隔（如每天、每周）重复抓取目标网站，是消费方主动、周期性发起请求的行为，符合 Poll 的定义。

**10.** The data quality framework includes six dimensions. List at least five dimensions, and for the scenario of "missing values occurring non-randomly," explain which dimension is affected and the specific harm to downstream analysis.

> [!tip]- Answer
> **六个数据质量维度**（至少列出五个）：
> 1. **Believable（可信性）** — 数据是否来自可靠来源、用户是否信任它
> 2. **Value-Added（增值性）** — 数据是否对业务决策有实际价值
> 3. **Relevant（相关性）** — 数据是否与分析目标相关
> 4. **Accurate（准确性）** — 数据是否正确反映现实
> 5. **Interpretable（可解释性）** — 数据是否有足够的文档/元数据供理解
> 6. **Accessible（可访问性）** — 数据是否易于获取和使用（第6个维度）
>
> **非随机缺失的危害：**
> 缺失值若**非随机（non-random）**发生，主要损害 **Accurate** 和 **Believable** 维度，并导致**代表性偏差（underrepresentation bias）**。例如：若问卷中收入越高的人越倾向于不填写收入字段，则删除这些缺失值后，样本中高收入群体严重不足，任何基于该数据的统计分析或模型都会系统性地低估高收入群体的行为特征，结论不可信。

**11.** Given the following HTML snippet, describe its structure and explain how a data engineer would locate the names and prices of all products when scraping this page.

```html
<div class="product-list">
  <div class="product-item">
    <span class="product-name">Wireless Earbuds Pro</span>
    <span class="product-price">¥299</span>
  </div>
  <div class="product-item">
    <span class="product-name">Portable Power Bank</span>
    <span class="product-price">¥89</span>
  </div>
</div>
```

> [!tip]- Answer
> **HTML 结构解析：**
> - 最外层 `<div class="product-list">` 是产品列表容器。
> - 内部每个 `<div class="product-item">` 代表一个产品条目。
> - 每个条目内有两个 `<span>`：`class="product-name"` 存储产品名称，`class="product-price"` 存储价格。
>
> **定位方式：**
> - **CSS Selector**：使用 `.product-item .product-name` 选取所有产品名称；`.product-item .product-price` 选取所有价格。
> - **XPath**：`//div[@class="product-item"]/span[@class="product-name"]` 选取所有名称节点，提取其 `text()` 内容。
> - 实际爬取时，先找到所有 `.product-item` 节点，再对每个节点分别提取子元素的文本，形成结构化的（名称, 价格）对。
