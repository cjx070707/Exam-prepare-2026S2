**8.** 以下 SQL 查询的作用是什么？

```sql
SELECT s.name, r.region_name
FROM Stations s, Regions r
WHERE ST_Contains(r.geom, s.geom)
  AND r.region_name = 'Greater Sydney';
```

## 三、应用题

**9.** 你需要查询数据库中距离悉尼市中心（坐标：`POINT(151.2093 -33.8688)`）10公里以内的所有餐厅。假设餐厅表 `restaurants` 有字段 `name` 和 `location`（geometry 类型，SRID 4326）：

写出 PostGIS SQL 查询。

**10.** 描述从获取含经纬度的 CSV 数据到执行空间查询的完整工作流程（至少5个步骤）。

---

## 参考答案

1. **B**
2. **B**
3. **B**
4. **C**
5. **B**
6. R-Tree 将空间对象用最小外接矩形（MBR）包围，并按层级组织这些矩形。查询时先用矩形过滤候选集（粗筛），再做精确空间计算（精筛），大幅减少需要精确计算的对象数量。
7.
   | 格式 | 特点 | 用途 |
   |------|------|------|
   | GeoJSON | JSON 格式，易与 Web API 集成 | Web 地图，API 数据交换 |
   | KML | XML 格式，Google 标准 | Google Earth 可视化 |
   | WKT | 文本格式，如 `POINT(151.2 -33.8)` | 数据库存储和调试 |
8. 这是一个空间 Join 查询：找出所有位于 "Greater Sydney" 区域内的气象站名称，以及它们所属的区域名称。`ST_Contains` 判断站点坐标是否在区域多边形内。
9.
```sql
SELECT name
FROM restaurants
WHERE ST_Distance(
    location,
    ST_GeomFromText('POINT(151.2093 -33.8688)', 4326)
) <= 10000;
```
（注：ST_Distance 在 geography 类型下单位为米，geometry 类型下单位为坐标系单位）
10. 完整工作流程：
    1. 获取含经纬度列的 CSV 数据
    2. 安装 PostGIS 扩展，创建含 geometry 字段的表
    3. 将经纬度转为 geometry 类型存入数据库（确认 SRID 为 4326）
    4. 建立 GiST 空间索引：`CREATE INDEX ON table USING GIST(geom)`
    5. 执行空间查询（距离、包含、相交等）
    6. 将结果导出为 GeoJSON 在地图上可视化
