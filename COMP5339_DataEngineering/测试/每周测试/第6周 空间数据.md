## Multiple Choice

**1.** A urban planning company is comparing two spatial data modelling approaches: one represents discrete objects such as buildings, roads, and parks; the other represents a continuous temperature distribution layer for heat island analysis. Which spatial data models do these correspond to, respectively?

- A. Field-based and Object-based
- B. Object-based and Field-based
- C. Both are Object-based, just at different granularities
- D. Both are Field-based, differing only in raster resolution

> [!tip]- Answer
> **答案：B**
> **Object-based 模型**将空间表示为离散、有边界的对象（建筑物、道路、公园），每个对象有独立属性。**Field-based 模型**将空间表示为连续的值域（如温度、高程、降雨量），整个研究区域每一点都有值。两者本质区别：前者是"有什么对象在这里"，后者是"这里的值是多少"。

**2.** A GIS team conducting land management analysis within Australia needs to legally match official cadastral data. Which coordinate reference system should they use, and why?

- A. WGS84 (SRID 4326), because it is the global GPS standard with the highest accuracy
- B. Mercator projection, because it preserves area and is suitable for land area calculations
- C. GDA94, because it is the official Australian survey coordinate system and aligns with local cadastral data
- D. Geocentric coordinates, because they use the Earth's centre as origin and reduce error

> [!tip]- Answer
> **答案：C**
> **GDA94**（Geocentric Datum of Australia 1994）是澳大利亚官方测量和地籍的标准坐标系，政府土地数据均以此为基准。WGS84 是 GPS 全球标准，与 GDA94 差约1米（在很多应用中可互换，但法定测量场景必须用 GDA94）。Mercator 投影不保面积而保角（等角投影），大范围面积计算会失真。

**3.** A global logistics company needs to calculate the length of shipping routes spanning multiple continents (thousands of kilometres). When storing routes in PostGIS, should engineers use the `geometry` or `geography` type, and why?

- A. `geometry`, because it is faster and more suitable for long-distance routes
- B. `geography`, because it is based on a spherical model and provides higher accuracy for large-scale distance calculations
- C. There is no difference; PostGIS automatically selects the appropriate model
- D. `geometry`, because the `geography` type does not support LineString

> [!tip]- Answer
> **答案：B**
> **`geometry`** 基于平面模型（笛卡尔坐标），距离单位取决于 SRID（如度），在小范围或已投影坐标系下计算快、够用。**`geography`** 基于球面（椭球体）模型，距离单位始终为米，适合跨大洲的大范围计算，不会因 Mercator 等投影的变形导致误差。核心原则：**范围越大 → 用 `geography`**。

**4.** The Nine-Intersection Model (DE-9IM) decomposes the spatial relationship between two geometries into nine combinations of Interior / Boundary / Exterior intersections. Which relationship describes "two polygons that share a boundary segment but whose interiors do not overlap"?

- A. Contains
- B. Disjoint
- C. Touches
- D. Overlaps

> [!tip]- Answer
> **答案：C**
> **Touches**（接触）：两个几何体的 Boundary 有交集，但 Interior 不相交。典型场景：两块相邻的土地地块，共享边界线但不重叠。**Disjoint**：完全不接触。**Overlaps**：Interior 有部分重叠（面积交集）。**Contains**：一个完全包含另一个的内部。九交叉模型的核心就是通过这9个 I/B/E 组合的 T/F 矩阵区分8种拓扑关系。

**5.** An engineer creates a GiST spatial index (R-Tree) on a table storing one million restaurant locations. Which of the following best describes how this index works?

- A. R-Tree sorts all points by longitude into a B-Tree, then filters by latitude in a second pass
- B. R-Tree wraps spatial objects in Minimum Bounding Rectangles (MBRs), organises them in a nested hierarchy, and uses MBR filtering first before precise geometric computation
- C. R-Tree divides the entire map into equal-sized grid cells and stores corresponding points in each cell
- D. R-Tree hashes spatial objects into fixed buckets for O(1) lookup

> [!tip]- Answer
> **答案：B**
> **R-Tree** 的核心思路：① 每个空间对象用 **MBR（Minimum Bounding Rectangle）** 近似；② MBR 按层级组织成树，父节点 MBR 包含所有子节点 MBR；③ 查询时从根节点开始，跳过与查询区域不相交的子树（**粗筛剪枝**），只对少量候选对象做精确几何计算（**精筛**）。这将时间复杂度从 O(n) 降至 O(log n)。选项 C 描述的是网格索引，D 是哈希索引，两者都不是 R-Tree。

**6.** A data engineer performs a Spatial Join between two datasets from different departments: Department A uses GDA94 (SRID 7844) and Department B uses WGS84 (SRID 4326). If no preprocessing is done, what is the most likely outcome?

- A. The query immediately throws an error; PostGIS refuses to execute operations with mismatched SRIDs
- B. Results appear normal, but spatial relationships are computed in the wrong coordinate frame, producing systematic errors
- C. PostGIS automatically detects both coordinate systems and converts in the background; results are correct
- D. Only distance calculations are affected; topological predicates such as Contains and Intersects are unaffected

> [!tip]- Answer
> **答案：B**
> PostGIS 在 SRID 不一致时**不保证自动转换**——某些版本会报错，某些版本静默地在混用坐标系的情况下计算，结果完全不可信（拓扑关系和距离都会出错）。正确做法是用 `ST_Transform()` 显式统一 SRID 后再运算。**考试核心点**：SRID 不统一是空间数据工程中最常见的"静默错误"来源，结果看似有输出，实则错误。

**7.** A mobile application needs to transmit user-annotated locations to a web backend in real time and display them on a map. Which spatial data format is most appropriate for this scenario?

- A. Shapefile, because it is the GIS industry standard with the best compatibility
- B. WKT, because its syntax is concise and file size is small
- C. KML, because it contains rich styling and annotation information
- D. GeoJSON, because it is JSON-based, can be embedded directly in REST API responses, and is natively parsed by web mapping libraries

> [!tip]- Answer
> **答案：D**
> **GeoJSON** 是 Web 场景的首选：① 基于 JSON，与现代 REST API 无缝集成；② Leaflet、Mapbox、Google Maps 等主流 Web 地图库原生支持；③ 轻量、人类可读。**Shapefile** 是多文件格式（.shp/.dbf/.prj等），不适合 API 传输。**KML** 是 XML 格式、Google Earth 专用，Web 集成复杂。**WKT** 是数据库内部文本格式，缺乏属性信息承载能力，不适合 API 响应。

**8.** What is the primary limitation of DuckDB's spatial extension compared to PostGIS?

- A. DuckDB does not support reading or writing GeoJSON format
- B. DuckDB has no dedicated Geography type; all spatial calculations default to a planar model (EPSG:4326) and cannot natively compute spherical distances
- C. DuckDB's spatial extension only supports Point and Polygon geometry types
- D. DuckDB's spatial index performance is more than 10× slower than PostGIS and is not suitable for production use

> [!tip]- Answer
> **答案：B**
> PostGIS 有 `geometry`（平面）和 `geography`（球面）两种类型，`geography` 可以做精确的球面距离计算，适合大范围全球数据。DuckDB 的空间扩展**只有 `geometry` 类型**，默认坐标系 EPSG:4326，所有计算基于平面模型——这在小范围（城市级别）可接受，但跨洲际的大范围距离计算会有系统性误差。DuckDB 空间扩展的定位是分析型场景的便捷工具，而非替代 PostGIS 的生产级空间数据库。

## Short Answer

**9.** Explain the difference between Spatial and Geospatial data, and describe the division of responsibilities between an SDBMS and a GIS. **(4 Marks)**

> [!tip]- Answer
> **Spatial vs Geospatial（2分）：**
> - **Spatial**（空间数据）：泛指任意坐标空间中的数据，坐标不必对应真实地理位置（如医学影像坐标、工厂平面图坐标）。
> - **Geospatial**（地理空间数据）：特指对应地球表面真实地理位置的数据，使用经纬度或地图投影坐标，与现实地理位置绑定。
>
> **SDBMS vs GIS 分工（2分）：**
> - **SDBMS（Spatial DBMS）**：负责后端存储、空间索引（R-Tree/GiST）、空间查询执行（如 ST_Contains、ST_Distance）——是"存储和检索引擎"，如 PostGIS。
> - **GIS（Geographic Information System）**：负责前端分析与可视化，将数据组织成图层（Layer），提供地图渲染、叠加分析、路径规划等功能——是"分析和展示平台"，如 QGIS、ArcGIS。
> - 两者常配合使用：GIS 从 SDBMS 查询数据，再做空间分析和可视化。

**10.** A bike-sharing company plans to analyse which areas have an excessive bike parking density (more than 500 bikes per square kilometre). Describe three engineering challenges involved in this spatial analysis and provide a concrete strategy to address each. **(6 Marks)**

> [!tip]- Answer
> **挑战一：坐标系不统一（2分）**
> - 问题：单车 GPS 数据（WGS84）与城市区域边界数据（可能是 GDA94 或本地投影）SRID 不一致，直接做 Spatial Join 结果错误。
> - 应对：在入库或查询时统一用 `ST_Transform()` 转换到同一坐标系，并在 pipeline 中做 SRID 校验。
>
> **挑战二：Spatial Join 计算代价高（2分）**
> - 问题：将数百万条停车记录与城市区域多边形做空间包含查询（ST_Contains），若无索引则为 O(n×m) 全表扫描，性能不可接受。
> - 应对：在几何列上建立 GiST 索引（R-Tree），先用 MBR 粗筛大幅减少候选对，再做精确几何计算。
>
> **挑战三：大范围距离/面积计算的精度问题（2分）**
> - 问题：若使用平面 `geometry` 类型计算城市区域面积，Mercator 等投影在高纬度或大范围会严重失真，导致密度计算错误。
> - 应对：对面积计算使用 `geography` 类型（球面模型），或先将数据投影到适合本地的等积投影（如澳大利亚 Albers 等积投影），再计算面积。

**11.** Explain the basic framework of the Nine-Intersection Model and use it to distinguish the three topological relationships: Disjoint, Touches, and Overlaps. **(5 Marks)**

> [!tip]- Answer
> **九交叉模型框架（2分）：**
> 将每个几何体分解为三个部分：**Interior（内部 I）**、**Boundary（边界 B）**、**Exterior（外部 E）**。对两个几何体 A 和 B，两两组合得到 9 个交集：I(A)∩I(B)、I(A)∩B(B)、…、E(A)∩E(B)，每个交集判断是否为空（T/F），形成 3×3 的 T/F 矩阵。不同矩阵模式对应不同的拓扑关系。
>
> **三种关系的区别（3分）：**
> - **Disjoint（不相交）**：A 和 B 完全分离，I(A)∩I(B)=∅，B(A)∩B(B)=∅，两者任何部分均无接触。
> - **Touches（接触）**：边界有交集（B(A)∩B(B)≠∅ 或 I∩B≠∅），但内部不重叠（I(A)∩I(B)=∅）。典型例子：两块相邻地块共享边界线。
> - **Overlaps（重叠）**：内部有面积交集（I(A)∩I(B)≠∅），但各自也有对方不覆盖的部分——即部分重叠，非完全包含。典型例子：两个有交集的服务覆盖区域。
