## Part 1 — 数据从哪来（W1 + W2）

> [!info] 问题链
> 数据在哪里？→ 文件 / 数据库 / 网页 / 流 → 不同来源用不同方式获取 → 数据到手了，大小有没有边界？→ 无界流数据必须切成有界批次才能处理

### 数据源分类 `★★`

> [!tip] 一个零售公司既有历史 CSV 订单、又有实时传感器数据——从哪取，用什么格式？

| 来源 | 例子 | 常见格式 |
|------|------|----------|
| **File Access** | 企业已有数据集，data.gov.au 下载 | CSV, Excel, XML, JSON, 图片 |
| **Database Access** | 生产系统数据库 | SQL 查询结果 |
| **Web / APIs** | 网页爬取，Web Service API | HTML, JSON, XML |
| **CDC / Logs** | 数据库变更日志，消息队列 | 流数据 |
| **Messages / Streams** | IoT 传感器，Pub/Sub 系统 | 实时流 |

<span style="color: #9b59b6">💡 **白话理解**：数据来源决定获取手段——静态文件直接下载，数据库用 SQL 查，网页用爬虫或 API，流数据要用消息系统（Kafka/MQTT）。</span>

<span style="color: #e67e22">*获取了数据，首先要问：这批数据是有终点的文件，还是永不停歇的流？——这决定后续怎么处理它。*</span>

### Bounded vs Unbounded Data `★★★`

> [!tip] 传感器每秒发一条温度读数，永远不会停——这种数据怎么处理？

| 类型 | 含义 | 例子 |
|------|------|------|
| **Bounded（有界）** | 有固定大小的数据集或批次 | 下载的文件，API 一次返回的结果 |
| **Unbounded（无界）** | 持续产生、没有终点的数据 | 传感器读数流，日志流 |

> 所有 Unbounded 数据在处理前必须转换成 Bounded（划定窗口或批次）

<span style="color: #9b59b6">💡 **白话理解**：Bounded 是"一袋苹果"（有头有尾）；Unbounded 是"自来水管"（不停流）。批处理系统只能处理袋装苹果——流数据必须先切成一段一段（窗口）才能批量计算。</span>

## Part 2 — Unix 命令行工具（W1） `★`

> [!info] 问题链
> 数据到手后，最快的初步探查方式是什么？→ Unix 命令行——一行命令就能看头几行、统计行数、过滤特定行

> **注意**：老师明确说不考 Unix 命令代码，以下为背景了解，重点在理解每个命令的作用。

> [!tip] 拿到一个 100 万行的 CSV，怎么快速看看前 5 行和总行数？

| 命令 | 用途 | 例子 |
|------|------|------|
| `cat` | 输出文件内容 | `cat data.csv` |
| `head` / `tail` | 看前/后 N 行 | `head -5 data.csv` |
| `sort` | 排序 | `sort data.csv` |
| `grep` | 过滤匹配行 | `grep "Sydney" data.csv` |
| `wc` | 统计行/词/字符数 | `wc -l data.csv` |
| `cut` | 按列切分 | `cut -d',' -f1,3 data.csv` |
| `sed` | 流式文本替换 | `sed -e 's/hello/world/g' file` |

**Piping（管道）**：把一个命令的输出直接作为下一个命令的输入

```bash
cat data.csv | sort | head -3
```

<span style="color: #9b59b6">💡 **白话理解**：Unix 中"一切皆文件"——命令输出是流，可以直接通过 `|` 传递，无需中间存储。像流水线：一个命令的输出直接喂给下一个命令。</span>

## Part 3 — 数据质量（W2）

> [!info] 问题链
> 数据拉进来了——但它干净吗？→ 先评估质量六维度 → 发现哪类问题 → 选对应清洗策略

### 数据质量六维度 `★★★`

> [!tip] 数据显示某产品上周销量 10 倍暴增，但你知道那周仓库关门了——这是哪类质量问题？

| 维度 | 含义 | 例子 |
|------|------|------|
| **Believable** | 数据是否可信 | 非高峰期销售数据突然暴增 |
| **Value-Added** | 数据是否有用 | 与转化率无关的流量数据 |
| **Relevant** | 数据是否相关 | 把已停产品数据混入当前分析 |
| **Accurate** | 数据是否准确 | 姓名地址不匹配 |
| **Interpretable** | 数据是否可理解 | 报告中充满未定义的缩写 |

<span style="color: #9b59b6">💡 **白话理解**：数据质量不只是"有没有缺失值"——一条准确但不相关的数据，对分析来说同样是噪声。六维度帮你全面评估数据是否"值得用"。</span>

<span style="color: #e67e22">*评估完质量，下一步是识别具体问题类型，对症下药处理。*</span>

### 常见数据问题类型 `★★★`

> [!tip] 年龄字段里有 0、有 999、有空值——三种情况分别是哪类问题，怎么处理？

| 问题 | 原因 | 处理方式 |
|------|------|----------|
| **Missing（缺失）** | 未填写、传输丢失 | `dropna()` 删除，或 `fillna()` 填充 |
| **Default（占位符）** | 用 0 / 99999 代替缺失 | 识别后替换为 NaN |
| **Incorrect（错误值）** | 传感器故障，输入错误 | 用领域知识检测，删除或修正 |
| **Inconsistent（不一致）** | 格式不统一，大小写混用 | 标准化，如统一小写 |

<span style="color: #9b59b6">💡 **白话理解**：Missing 是"空格子"；Default 是"占位符假装有值"（比如用 99999 表示未知年龄）；Incorrect 是"填了但填错了"；Inconsistent 是"格式乱了"（如 "sydney" / "Sydney" / "SYDNEY" 同一城市三种写法）。</span>

## Part 4 — Pandas 数据处理（W2）

> [!info] 问题链
> 发现了数据问题，用什么工具清洗？→ Pandas：读取、检查、填充、替换、转类型

> **注意**：老师明确说不考 Python 代码阅读，Pandas 方法名作为概念了解即可。

### 读取与检查 `★★`

> [!tip] 拿到一个 CSV，怎么快速知道有多少行、哪些列有缺失值？

```python
import pandas as pd
df = pd.read_csv('data.csv')
df.shape          # 行列数
df.info()         # 每列类型 + 非空计数
df.describe()     # 数值列统计摘要
```

### 缺失值处理 `★★★`

> [!tip] `dropna` 和 `fillna` 分别在什么情况下选？

```python
df.dropna()                        # 删除含缺失值的行
df.fillna(0)                       # 用 0 填充缺失值
df['col'].fillna(0).astype(int)    # 填充后转类型
df['col'].replace('old', 'new')    # 替换特定值（处理 Default 问题）
```

<span style="color: #9b59b6">💡 **白话理解**：`dropna` = 删掉整行（缺失太多时用）；`fillna` = 用某个值填坑（缺失少、能估计时用）；`replace` = 把占位符（如 99999）换成 NaN，再后续处理。</span>

### 类型转换 `★★`

> [!tip] 为什么把 float 列转成 int 前必须先处理缺失值？

```python
df['col'] = df['col'].astype(float)
# 注意：int 不支持 NaN，转 int 前必须先处理缺失值
```

### 数据可视化（快速检验数据质量） `★`

- **Bar Chart**：分类/名义数据
- **Histogram**：数值分布
- **Scatter Plot**：两个连续变量的关系

<span style="color: #9b59b6">💡 **白话理解**：可视化是数据质量的"X光"——一眼看出分布异常、极端值、意外的空值模式。</span>

## Part 5 — Web 爬虫与 API（W4）

> [!info] 问题链
> 数据在网页上，没有 API 直接提供——怎么把它结构化地取出来？→ 理解 HTML 树形结构 → 用 CSS Selector / XPath 定位元素 → 提取数据 → 对比直接调用 API 的方式

### HTML 基础 `★★★`

> [!tip] 网页上的表格数据，怎么告诉爬虫"我要这个格子"而不是旁边那个？

- HTML 是**树形结构**（DOM 树）
- 标签有类型（`<div>`, `<table>`, `<h1>`）、class 属性、id 属性
- Web 爬虫的核心：找到正确的 HTML 元素提取数据

<span style="color: #9b59b6">💡 **白话理解**：HTML 像一棵家族树——`<html>` 是祖先，`<body>` 是父节点，`<table>` 是子节点，`<td>` 是孙节点。CSS Selector 就是家族地址："找到 class='data' 的 table 下第二个 tr 里的第一个 td"。</span>

<span style="color: #e67e22">*能读懂 HTML 结构之后，下一步是用 HTTP 发请求拿到网页内容。*</span>

### HTTP 请求 `★★`

> [!tip] `GET` 和 `POST` 的区别是什么？什么时候用哪个？

- **GET**：从服务器取数据，参数拼在 URL 里
- **POST**：向服务器提交数据（如登录），参数在请求体里
- 状态码：`200` = 成功，`404` = 找不到，`403` = 禁止访问

<span style="color: #e67e22">*拿到 HTML 内容后，需要解析它——找到目标数据所在的标签。*</span>

### BeautifulSoup 解析 HTML `★★`

> [!tip] 一个页面有 5 个 `<table>`，怎么只取第二个里面的数据？

```python
from bs4 import BeautifulSoup

content = BeautifulSoup(response.text, 'html5lib')

content.find('h3')                    # 找第一个 h3
content.find_all('table')             # 找所有 table → 返回列表
content.find(id='results')            # 按 id 找（唯一）
content.find_all('table', 'data')     # 按标签 + class 找
content.select('#ship.data')          # 用 CSS selector
```

<span style="color: #9b59b6">💡 **白话理解**：`find` = "找第一个"；`find_all` = "找全部，返回列表"；`select` = "用 CSS 地址精确定位"。`find_all('table')[1]` 就是第二个表格。</span>

<span style="color: #e67e22">*BeautifulSoup 解析后，CSS Selector 是定位元素的核心语法——考试最常考这个。*</span>

### CSS Selector 规则 `★★★`

> [!tip] 如何用一行 CSS Selector 选到 class 为 "price" 的 span 里的文字？

| 选择器 | 含义 | 例子 |
|--------|------|------|
| `tag` | 按标签类型 | `table` |
| `tag.class` | 按标签 + class | `table.data` |
| `#id` | 按 id | `#results` |
| `parent child` | 后代选择器 | `div.container span` |
| `tag:nth-child(n)` | 第 n 个子元素 | `li:first-child` |

<span style="color: #9b59b6">💡 **白话理解**：CSS Selector 像 GPS 地址——`table.data td` 意思是"找 class='data' 的 table 里的所有 td 格子"。Quiz 2 考过 HTML 代码阅读，这是重点。</span>

<span style="color: #e67e22">*网页爬取不稳定（页面结构可能随时改）——如果数据提供方有 API，优先用 API。*</span>

### Web API `★★`

> [!tip] API 和直接爬网页相比，有什么优势？什么情况只能爬？

- API 返回**结构化数据**（JSON 或 XML），不含 HTML 展示信息
- **更稳定**：网页改版不影响 API 响应格式
- **更适合自动化**：不需要解析 HTML，直接 `.json()` 获得 Python 字典
- 很多 API 需要 **API Key** 认证（控制访问频率和权限）

<span style="color: #9b59b6">💡 **白话理解**：API 是"官方数据出口"——你拿到的是干净的 JSON，不用在 HTML 里翻来覆去找数据。爬虫是"抄黑板"，API 是"直接拿讲义"。只有没有 API 时才考虑爬网页。</span>

### 工具对比 `★★`

> [!tip] 网站用 JavaScript 动态加载内容，requests 拿不到数据，该用什么工具？

| 工具 | 适用场景 |
|------|----------|
| `requests` + `BeautifulSoup` | 单页面/少量页面爬取 |
| `Scrapy` | 多页面自动跟链爬取（spider） |
| `Selenium` | 含 **JavaScript** 的动态网页 |
| `pd.read_html()` | 直接提取 HTML 表格 |
| Google Sheets `ImportHTML()` | 快速一次性导入网页表格 |

<span style="color: #9b59b6">💡 **白话理解**：`requests` 只能拿到服务器返回的原始 HTML，JS 动态生成的内容它看不到；`Selenium` 会真正打开浏览器执行 JS，所以能拿到动态内容。</span>
