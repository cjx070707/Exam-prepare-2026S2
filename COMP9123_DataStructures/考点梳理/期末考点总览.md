## 0. 期末常见题型（你要练的能力）

- **Describe in your own words**：能用 3–6 句把 main idea 说清
- **Write structured pseudocode**：能写成“函数/步骤/循环/条件”的结构
- **Argue correctness**：会写最小必要证明（不追求花哨）
- **Analyse running time**：会逐行/逐结构算 Big-O，能说清“谁主导”

## 1. 四道大题（Practice Exam 对标）

### P1 — ADT / 数据结构设计（W1–W3）

**典型问法**：给一套操作 + 复杂度约束（例如 `create/set/get O(1)`, `increase-size O(n)`），让你设计实现。

**你要写出来的得分点**：

- **结构描述**：存什么字段、用什么表示（array of rows / two stacks / etc.）
- **step-by-step**：每个 operation 做什么
- **复杂度解释**：为什么能做到题目要求；为什么别的直觉方案会超时

**快速联想**：

- `复习笔记/第一组 线性数据结构.md`：ADT vs DS；Big-O；数组/链表权衡
- `测试/考试模拟题/老师给考试题 1.md`：P1 折叠答案模板

### P2 — 图算法设计 + 正确性证明（W8–W9）

**典型问法**：给定义，要求你设计多项式算法，并证明最优/正确，再分析复杂度。

**你要写出来的得分点**：

- **Main idea**：一句话抓住“要优化什么”（例如最短 swapping sequence）
- **关键构造**：每一步怎么保证合法（例如加边形成环，再删某边保持树）
- **Correctness**：不求长篇，至少写清 2 点：
  - 每步保持不变式（还是 spanning tree）
  - 目标量单调减少（步数最少的下界匹配）
- **复杂度**：每一步做什么图操作（BFS/DFS/Union-Find/PQ），乘以步数上界

**快速联想**：

- `复习笔记/第三组 哈希与图.md`：图表示、BFS/DFS、MST、Dijkstra 的依赖结构
- `测试/第三组 哈希与图 测试.md`：操作追踪 + 复杂度 + 对比题型

### P3 — 综合技能题（MST / BST / AVL）（W4–W5, W9）

#### (a) MST（W9）

**会做**：

- 用 **Kruskal/Prim** 画 MST（知道“避免成环”的原则）
- 会说清（若要求）复杂度：`Kruskal = sort edges + Union-Find`；`Prim = PQ`

#### (b) BST（W5）

**会做**：

- 写 **pre/in/post-order**（不会写错顺序）
- 删除结点三情形（0/1/2 children）：
  - 2 children：用 **inorder successor/predecessor** 替换再删

#### (c) AVL “增广”题（W4/W5 思路，考场变式）

**会写得分点**：

- **AVL 基本**：旋转保持高度平衡 → worst-case `O(log n)`
- **Augment 的套路**：每个结点多存一个字段（例如 `size`），插入/旋转时同步更新
- **按名次查询**：用 `left.size` 决定向左/返回/向右，`O(log n)`

**快速联想**：

- `复习笔记/第二组 树与有序结构.md`：BST 删除；AVL 旋转与维护高度
- `测试/第二组 树与有序结构 测试.md`：BST/AVL 对比、range query
- `测试/考试模拟题/老师给考试题 1.md`：P3 折叠答案

### P4 — 分治 + 正确性 + 复杂度（W11）

**典型问法**：设计 `O(n log n)` 分治算法，证明不会漏/重数，再写 Big-O。

**你要写出来的得分点**：

- **Split / Solve / Combine**：分成左右；子问题答案；跨区间如何统计
- **Correctness 结构**：所有对要么在左、要么在右、要么跨边界（分类不重不漏）
- **复杂度**：写出递推式 `T(n)=2T(n/2)+O(n)` → `O(n log n)`

**快速联想**：

- `复习笔记/第四组 算法设计范式.md`：D&C 模板；merge sort；递推求解
- `测试/第四组 算法设计范式 测试.md`：递推/unrolling、分治追踪

## 2. ★★★ 高频考点（按主题速查）

> 这一节只列“上考场最常用”的 ★★★（以及直接服务期末题型的少量 ★★）。详细星级审计见 `更新进度.md`。

### W1–W3（线性 + 分析）

- **ADT vs Data Structure**：题目问的是“接口/行为”还是“实现/表示”
- **Big-O**：嵌套循环、递归、摊还（rehash/resize 的直觉）
- **Singly/Doubly Linked List**：插入/删除的指针顺序；给定结点删除为什么双链表 O(1)
- **Stack / Queue**：操作追踪；场景（括号匹配/BFS）

### W4–W6（树与有序结构）

- **Traversals**：pre/in/post 顺序绝不写错
- **BST delete**：successor/predecessor 的使用
- **AVL**：失衡点 z/y/x；单旋/双旋；为什么高度 O(log n)
- **Range query（BST/AVL）**：`O(log n + s)` 结构解释（s=输出数）

### W7–W9（哈希与图）

- **Hash Table basics**：hash code + compression；load factor；rehash 的动机
- **Linear Probing**：DEFUNCT；get/put/remove 追踪
- **Cuckoo vs Linear**：worst-case lookup vs insertion rehash trade-off
- **Graph rep**：AdjList vs AdjMatrix 选型（稀疏/稠密）
- **BFS/DFS**：用什么结构（Queue/Stack）；典型用途（连通/最短路）
- **Dijkstra**：为什么需要 PQ；负权边不行的直觉
- **MST**：Kruskal/Prim 的核心思想 + 依赖结构（UF/PQ）

### W10–W12（范式）

- **Greedy proof**：exchange / stays-ahead；反例怎么写
- **D&C**：split/merge；`T(n)=aT(n/b)+f(n)` 的求解直觉（unrolling/递推树）
- **Recurrences**：写得出、解得出（至少能说明主导项）
- **Randomization**：Fisher–Yates 追踪；Skip List 的“期望 O(log n)”直觉
