**Question 1.** **(5 Marks)** Consider the following binary tree:

```
          15
        /    \
       8      25
      / \    /  \
     4  12  20   30
    /  /   /  \
   2  10  18  22
```

**(a)** Write the sequence of nodes visited by each of the following traversals. **(3 Marks)**
- Pre-order traversal
- In-order traversal
- Post-order traversal

**(b)** Explain why the in-order traversal of this tree produces a sorted sequence. What structural property of the tree guarantees this? **(2 Marks)**

> [!note]- Answer
> **(a) 三种遍历结果**
>
> **Pre-order（根→左→右）**：15, 8, 4, 2, 12, 10, 25, 20, 18, 22, 30
>
> 规则：先访问节点自身，再递归遍历左子树，再递归遍历右子树。
>
> **In-order（左→根→右）**：2, 4, 8, 10, 12, 15, 18, 20, 22, 25, 30
>
> 规则：先递归遍历左子树，再访问节点自身，再递归遍历右子树。
>
> **Post-order（左→右→根）**：2, 4, 10, 12, 8, 18, 22, 20, 30, 25, 15
>
> 规则：先递归遍历左子树，再递归遍历右子树，再访问节点自身。
>
> **(b) 为什么 in-order 得到排序序列**
>
> 这棵树满足 **BST 性质（Binary Search Tree property）**：对于任意节点 v，左子树中所有节点的 key < key(v) < 右子树中所有节点的 key。
>
> In-order 遍历先递归处理左子树（所有更小的 key），再处理当前节点，再处理右子树（所有更大的 key）。BST 性质保证每一层递归都满足这个"小→中→大"的关系，所以整体输出恰好是升序排列。

**Question 2.** **(6 Marks)** Starting with an empty BST, perform the following operations **in order** and show the tree after each step:

**(a)** Insert keys in this order: 20, 10, 30, 5, 15, 25, 35. Draw the resulting BST. **(2 Marks)**

**(b)** Delete key 20 from the tree you built in (a). Show the step-by-step process, clearly stating which deletion case applies and identifying the inorder successor if relevant. Draw the final tree. **(2 Marks)**

**(c)** Now delete key 10 from the tree resulting from (b). Again show the process and the final tree. **(2 Marks)**

> [!note]- Answer
> **(a) 依次插入后的 BST**
>
> 插入 20 → root = 20
> 插入 10 → 10 < 20，放左
> 插入 30 → 30 > 20，放右
> 插入 5 → 5 < 20 < 10，放 10 的左
> 插入 15 → 15 < 20，15 > 10，放 10 的右
> 插入 25 → 25 > 20，25 < 30，放 30 的左
> 插入 35 → 35 > 20，35 > 30，放 30 的右
>
> ```
>          20
>        /    \
>      10      30
>     /  \    /  \
>    5   15  25   35
> ```
>
> **(b) 删除 key 20**
>
> 节点 20 有两个 internal children → **Case 2**（两个内部子节点）。
>
> Step 1: 找 20 的 **inorder successor**（右子树中最小的节点）= 25（从 30 往左走，25 没有左子，所以 25 就是最小的）。
>
> Step 2: 将 25 的 key 复制到 20 的位置。
>
> Step 3: 删除原来的 25 节点。25 有一个 external left child → Case 1：删除 25 和它的 external child，无需提升（25 本身没有 internal child 需要提升。30 的 left 变为 null/external）。
>
> ```
>          25
>        /    \
>      10      30
>     /  \       \
>    5   15       35
> ```
>
> **(c) 删除 key 10**
>
> 节点 10 有两个 internal children → **Case 2**。
>
> Step 1: 找 10 的 inorder successor = 15（右子树中最小的节点，15 本身就是右子节点且无左子）。
>
> Step 2: 将 15 的 key 复制到 10 的位置。
>
> Step 3: 删除原来的 15 节点。15 没有 internal child（Case 1），直接移除。
>
> ```
>          25
>        /    \
>      15      30
>     /          \
>    5            35
> ```
>
> **总结**：BST 删除的关键——有两个孩子时，用 inorder successor（右子树最左节点）替代被删节点，然后递归删除 successor（必定是 Case 1，因为 successor 不可能有左 internal child）。

**Question 3.** **(5 Marks)** Consider the following pseudocode that operates on a BST:

```
def mystery(v, k1, k2):
    if v is external then
        return 0
    if key(v) > k2 then
        return mystery(v.left, k1, k2)
    else if key(v) < k1 then
        return mystery(v.right, k1, k2)
    else
        return 1 + mystery(v.left, k1, k2) + mystery(v.right, k1, k2)
```

**(a)** Given a BST containing keys {2, 5, 9, 11, 12, 20, 22, 24, 26}, what does `mystery(root, 8, 22)` return? Trace through the key comparisons. **(2 Marks)**

**(b)** Describe in one sentence what this function computes in general. **(1 Mark)**

**(c)** Analyze the worst-case time complexity of this function in terms of the tree height h and the output size s (number of keys in [k1, k2]). Justify your answer. **(2 Marks)**

> [!note]- Answer
> **(a) 追踪 mystery(root, 8, 22)**
>
> 假设 BST 结构如下（按讲义中的树结构）：
> ```
>          12
>        /    \
>       5      24
>      / \    /  \
>     2   9  20   26
>            / \
>           11  22
> ```
>
> - mystery(12, 8, 22): 8 ≤ 12 ≤ 22 → 1 + mystery(5,...) + mystery(24,...)
> - mystery(5, 8, 22): 5 < 8 → mystery(9,...)
> - mystery(9, 8, 22): 8 ≤ 9 ≤ 22 → 1 + mystery(2,...) + mystery(null)
>   - mystery(2, 8, 22): 2 < 8 → mystery(null) → return 0
>   - right of 9 is external → return 0
>   - 结果：1 + 0 + 0 = 1
> - mystery(5) 返回：1
> - mystery(24, 8, 22): 24 > 22 → mystery(20,...)
> - mystery(20, 8, 22): 8 ≤ 20 ≤ 22 → 1 + mystery(11,...) + mystery(22,...)
>   - mystery(11, 8, 22): 8 ≤ 11 ≤ 22 → 1 + mystery(ext) + mystery(ext) = 1
>   - mystery(22, 8, 22): 8 ≤ 22 ≤ 22 → 1 + mystery(ext) + mystery(ext) = 1
>   - 结果：1 + 1 + 1 = 3
> - mystery(24) 返回：3
> - mystery(12) = 1 + 1 + 3 = **5**
>
> 返回值 **5**。范围 [8, 22] 内的 key 有：9, 11, 12, 20, 22，恰好 5 个。
>
> **(b) 功能描述**
>
> 该函数计算 BST 中 key 落在 [k1, k2] 范围内的节点总数（即 range count query）。
>
> **(c) 复杂度分析**
>
> 该算法的结构与 range search 完全一致（讲义中的 range query algorithm），只是输出计数而非收集节点。
>
> 访问的节点分为两类：
> - **Boundary nodes**（在 k1 或 k2 搜索路径上的节点）：最多 2h 个
> - **Inside nodes**（key 在 [k1, k2] 范围内但不在搜索路径上）：≤ s 个
>
> 每个访问的节点做 O(1) 工作（一次比较 + 可能的加法），因此：
>
> **Worst-case 时间复杂度：O(s + h)**
>
> 其中 s = 范围内的 key 数量，h = 树高。如果是 AVL 树，h = O(log n)。

**Question 4.** **(4 Marks)** A startup is building a product catalog system. They need to store (product_id, product_info) pairs and support the following operations:
- Look up a product by ID
- Add new products
- Remove discontinued products
- Find all products with IDs in a given range (e.g., all products with ID between 1000 and 2000)

**(a)** Compare three possible implementations — unsorted array, sorted linked list, and AVL tree — in terms of time complexity for each operation. Present your answer as a table. **(2 Marks)**

**(b)** Which implementation would you recommend and why? Consider both average-case performance and the range query requirement. **(2 Marks)**

> [!note]- Answer
> **(a) 三种实现的复杂度对比**
>
> | 操作 | Unsorted Array | Sorted Linked List | AVL Tree |
> |------|----------------|-------------------|----------|
> | Look up (get) | O(n) 线性搜索 | O(n) 即使有序也需遍历（无随机访问） | O(log n) BST 搜索 |
> | Add (put) | O(1) 直接尾部添加；如果需要查重则 O(n) | O(n) 找到有序插入位置 | O(log n) 搜索 + 插入 + 旋转 |
> | Remove | O(n) 搜索 + 移动元素 | O(n) 搜索节点（已知节点 O(1) 删除） | O(log n) 搜索 + 删除 + 可能多次旋转 |
> | Range query [k1, k2] | O(n) 遍历全部 + 过滤 | O(n) 遍历到 k1 位置 + 收集到 k2 | O(s + log n) s = 输出大小 |
>
> **(b) 推荐方案**
>
> **推荐 AVL Tree**，理由如下：
>
> 1. **单点操作**：所有操作（get/put/remove）都是 O(log n)，远优于 unsorted array 和 sorted linked list 的 O(n)。
>
> 2. **范围查询**：AVL Tree 支持 O(s + log n) 的 range query（s 是输出大小），这是该场景的关键需求。Unsorted array 必须扫描全部 O(n) 元素；sorted linked list 虽然数据有序，但找到范围起点就需要 O(n) 遍历。
>
> 3. **权衡**：AVL Tree 的实现复杂度较高（需要维护平衡和旋转逻辑），但作为 Sorted Map 的标准实现，它在所有维度上都表现均衡且最优。
>
> Unsorted array 只在"几乎不查询、只追加"的场景（如日志）有优势；sorted linked list 在此场景下没有任何操作优于 AVL Tree。

**Question 5.** **(5 Marks)** A hospital emergency department uses a min-heap to manage patient triage. Patients are assigned a priority key (1 = most urgent, higher = less urgent).

The current heap state is:

```
          2
        /   \
       5     3
      / \   /
     8   7  6
```

**(a)** A new patient arrives with priority 1. Show the heap after inserting key 1, including each swap performed during upheap. **(2 Marks)**

**(b)** After the insertion in (a), the most urgent patient (root) is called in for treatment. Perform `remove_min()` and show the heap after the operation, including each swap performed during downheap. **(2 Marks)**

**(c)** The hospital considers two alternative data structures for this triage system: an unsorted list and a sorted list. Compare the time complexity of `insert` and `remove_min` for all three implementations (unsorted list, sorted list, heap). Which is best suited for the ER scenario where both operations are frequent? **(1 Mark)**

> [!note]- Answer
> **(a) 插入 key 1 + Upheap**
>
> Step 1: 将 1 放到下一个可用位置（complete binary tree 最底层最左空位 = 8 的左 child）：
> ```
>          2
>        /   \
>       5     3
>      / \   /  \
>     8   7  6   (无)
>    /
>   1
> ```
> 等等——实际上 complete binary tree 要求最底层从左到右填满。当前有 6 个节点，下一个位置是 index 6 = 节点 3 的右 child。
>
> Step 1: 将 1 放在 3 的右 child 位置：
> ```
>          2
>        /   \
>       5     3
>      / \   / \
>     8   7  6   1
> ```
>
> Step 2: Upheap — 比较 1 和 parent(3)：1 < 3 → 交换
> ```
>          2
>        /   \
>       5     1
>      / \   / \
>     8   7  6   3
> ```
>
> Step 3: 比较 1 和 parent(2)：1 < 2 → 交换
> ```
>          1
>        /   \
>       5     2
>      / \   / \
>     8   7  6   3
> ```
>
> Step 4: 1 已经是 root，upheap 结束。
>
> **最终 Heap**：
> ```
>          1
>        /   \
>       5     2
>      / \   / \
>     8   7  6   3
> ```
>
> **(b) remove_min() + Downheap**
>
> Step 1: 移除 root (key=1)，用 last node (key=3, index 6) 替换 root：
> ```
>          3
>        /   \
>       5     2
>      / \   /
>     8   7  6
> ```
>
> Step 2: Downheap — 比较 3 和 children (5, 2)：min child = 2 < 3 → 交换 3 和 2
> ```
>          2
>        /   \
>       5     3
>      / \   /
>     8   7  6
> ```
>
> Step 3: 比较 3 和 children (6)：min child = 6 > 3 → 停止（3 ≤ 所有 children）。
>
> **最终 Heap**：
> ```
>          2
>        /   \
>       5     3
>      / \   /
>     8   7  6
> ```
>
> 注意：结果恰好恢复了原始的 heap 状态。
>
> **(c) 三种实现对比**
>
> | 实现 | insert | remove_min |
> |------|--------|------------|
> | Unsorted List | O(1) | O(n) |
> | Sorted List | O(n) | O(1) |
> | **Heap** | **O(log n)** | **O(log n)** |
>
> **Heap 最适合急诊室场景**。急诊室中 insert（新病人到达）和 remove_min（叫最紧急的病人）都频繁发生。Unsorted List 的 remove_min 是 O(n)——当病人很多时太慢；Sorted List 的 insert 是 O(n)——每次新病人到达都需要找正确位置。只有 Heap 能让两种操作都在 O(log n) 内完成，在两者都频繁的场景下提供最佳的**综合性能**。
