**Question 1.** Given the singly linked list `A → B → C → D → ∅` with a `head` pointer pointing to `A`, perform the following operations in sequence. For each step, draw the resulting linked list state and show any pointer changes. **(4 Marks)**

(a) Insert node `X` at position 2 (0-indexed). **(1 Mark)**

(b) Delete the node at the beginning. **(1 Mark)**

(c) Insert node `Y` at the end. **(1 Mark)**

(d) Delete the node at the end. **(1 Mark)**

> [!note]- Answer
> **(a) Insert X at position 2**
>
> 遍历到 position 1（即节点 B），执行：
> 1. `new_node.next = B.next` → X.next = C
> 2. `B.next = new_node` → B.next = X
>
> 结果：`A → B → X → C → D → ∅`
>
> **(b) Delete from beginning**
>
> `head = head.next` → head 指向 B
>
> 结果：`B → X → C → D → ∅`
>
> **(c) Insert Y at the end**
>
> 遍历到最后一个节点 D，`D.next = new_node`
>
> 结果：`B → X → C → D → Y → ∅`
>
> **(d) Delete from the end**
>
> 遍历到倒数第二个节点 D（`current.next.next == None` 时停），`D.next = None`
>
> 结果：`B → X → C → D → ∅`

**Question 2.** You are given a stack (initially empty) and a queue (initially empty). Perform the following interleaved operations and fill in the table. Show the state of each data structure and the return value after each operation. **(4 Marks)**

| Step | Operation | Stack State | Queue State | Return Value |
|------|-----------|-------------|-------------|--------------|
| 1 | push(8) | | | |
| 2 | enqueue(3) | | | |
| 3 | push(5) | | | |
| 4 | enqueue(1) | | | |
| 5 | pop() | | | |
| 6 | dequeue() | | | |
| 7 | top() | | | |
| 8 | enqueue(pop()) | | | |

> [!note]- Answer
>
> | Step | Operation | Stack State | Queue State | Return Value |
> |------|-----------|-------------|-------------|--------------|
> | 1 | push(8) | [8] | () | - |
> | 2 | enqueue(3) | [8] | (3) | - |
> | 3 | push(5) | [8, 5] | (3) | - |
> | 4 | enqueue(1) | [8, 5] | (3, 1) | - |
> | 5 | pop() | [8] | (3, 1) | 5 |
> | 6 | dequeue() | [8] | (1) | 3 |
> | 7 | top() | [8] | (1) | 8 |
> | 8 | enqueue(pop()) | [] | (1, 8) | 8 (popped) |
>
> Step 8 拆解：先执行 `pop()` 得到 8，再执行 `enqueue(8)`。栈变空，8 进入队列尾部。
> 注意栈是 LIFO（push/pop 都在栈顶），队列是 FIFO（enqueue 在队尾，dequeue 在队头）。

**Question 3.** Analyse the time complexity of the following pseudocode fragments using Big-O notation. Show your reasoning. **(4 Marks)**

**(a)** **(2 Marks)**

```
sum ← 0
for i ← 1 to n do
    j ← n
    while j > 1 do
        sum ← sum + j
        j ← j / 2
    end while
end for
```

**(b)** **(2 Marks)**

```
for i ← 1 to n do
    for j ← 1 to i do
        print(i, j)
    end for
end for
```

> [!note]- Answer
> **(a) O(n log n)**
>
> - 外层 `for` 循环执行 n 次。
> - 内层 `while` 循环：j 从 n 开始，每次除以 2，直到 j ≤ 1。执行次数 = log₂(n)。
> - 总计：n × log n = **O(n log n)**
>
> **(b) O(n²)**
>
> - 当 i = 1 时，内层执行 1 次；i = 2 时执行 2 次；... i = n 时执行 n 次。
> - 总执行次数 = 1 + 2 + 3 + ... + n = n(n+1)/2 = **O(n²)**
>
> 这是经典的等差数列求和，忽略常数项和低阶项后为 O(n²)。

**Scenario:** A music streaming app needs to manage a playlist. Users can:
- Add a song to the **end** of the playlist
- Skip (remove) the **currently playing** song (always the first in the list)
- Jump to a song at a specific position in the playlist and remove it
- Undo the last skip (restore the most recently skipped song back to the front)

**Question 4.** **(5 Marks)**

**(a)** For each of the four operations above, identify which ADT or data structure from the course (array, singly linked list, doubly linked list, stack, queue) would be most suitable. You may use more than one. Justify your choices by stating the time complexity achieved for each operation. **(3 Marks)**

**(b)** A colleague proposes using a single array-based list for everything. Explain why this would be problematic, referencing the specific operations that suffer. **(2 Marks)**

> [!note]- Answer
> **(a) 推荐组合：Queue（或 Singly Linked List）+ Stack**
>
> | 操作 | 推荐结构 | 复杂度 | 理由 |
> |------|----------|--------|------|
> | Add song to end | Queue / Linked List | O(1) 若有 tail 指针；O(n) 若无 | `enqueue` 天然在队尾添加 |
> | Skip (remove first) | Queue / Linked List | O(1) | `dequeue` / `head = head.next` |
> | Jump to position i & remove | Linked List | O(n) | 需要遍历到 position i-1 修改指针 |
> | Undo last skip | Stack | O(1) | 把跳过的歌 push 到 undo stack；undo 时 pop 并插回 head |
>
> 核心设计：主播放列表用 **singly linked list**（支持头部 O(1) 删除 + 尾部插入），配一个辅助 **stack** 记录跳过的歌（LIFO 顺序保证 undo 恢复最近一次跳过的歌）。
>
> 更优方案：若需要频繁 jump to position（中间删除），可用 **doubly linked list**——但只有在已持有节点引用时才有 O(1) 优势；按 index 访问仍需 O(n) 遍历。
>
> **(b) Array-based list 的问题**
>
> - **Skip (remove first)**：删除头部后需要将所有后续元素左移一位 → **O(n)**，播放列表越长越慢。
> - **Jump to position & remove**：同样需要移位 → **O(n)**。
> - **Undo**：数组本身没有 LIFO 语义，需要额外记录"最近跳过的歌"，仅用数组无法优雅实现。
> - **Add to end**：当数组满时无法动态扩展（固定大小限制），或需要 O(n) 复制到更大数组。
>
> 根本问题：数组的 `add(0, e)` 和 `remove(0)` 都是 O(n)，而播放列表的核心操作恰好是频繁的头部删除（skip）。Linked list 头部删除 O(1)，天然适合。

**Question 5.** You are designing a simple text editor that supports two features:
- **Type a character**: append a character to the end of the current text
- **Backspace**: delete the most recently typed character

The editor also needs an **undo** feature: pressing Ctrl+Z reverses the last action (if the last action was "type 'a'", undo removes 'a'; if the last action was "backspace", undo restores the deleted character).

**(8 Marks)**

**(a)** Design a data structure solution to support all three operations (`type`, `backspace`, `undo`) efficiently. Describe your solution in plain English, specifying which data structures you use and how each operation works. **(4 Marks)**

**(b)** Analyse the time complexity of `type`, `backspace`, and `undo`. **(2 Marks)**

**(c)** Now suppose the editor must also support `redo` — pressing Ctrl+Y re-applies the last undone action. Explain how your design needs to change and what happens to the redo history when the user types a new character after undoing. **(2 Marks)**

> [!note]- Answer
> **(a) 设计方案：两个栈**
>
> - **Text Stack**（主栈）：存储当前文本内容，栈顶是最后一个字符。
> - **Undo Stack**（撤销栈）：存储历史操作记录，每条记录是 `(action_type, character)` 的元组。
>
> 操作流程：
>
> | 操作 | 步骤 |
> |------|------|
> | `type(c)` | push `c` onto Text Stack；push `("type", c)` onto Undo Stack |
> | `backspace` | pop from Text Stack 得到 `c`；push `("backspace", c)` onto Undo Stack |
> | `undo` | pop from Undo Stack 得到 `(action, c)`；if action == "type" → pop from Text Stack（撤销添加）；if action == "backspace" → push `c` onto Text Stack（恢复删除的字符）|
>
> 两个栈的 LIFO 特性保证 undo 总是撤销最近一次操作。
>
> **(b) 复杂度分析**
>
> | 操作 | 时间 | 空间 |
> |------|------|------|
> | `type(c)` | O(1) | push 两次 |
> | `backspace` | O(1) | pop + push |
> | `undo` | O(1) | pop + 条件 push/pop |
>
> 所有操作都是 O(1)，因为栈的 push/pop 都是常数时间。总空间 O(n)，其中 n 是操作历史长度。
>
> **(c) 支持 Redo**
>
> 新增第三个栈 **Redo Stack**：
> - `undo` 操作时：把被撤销的操作 push 到 Redo Stack（而非丢弃）
> - `redo` 操作时：pop from Redo Stack，重新执行该操作
> - **关键**：当用户在 undo 之后输入新字符（`type`），必须**清空 Redo Stack**。因为新输入改变了文本状态，之前撤销的操作已经不再有意义——这与所有主流编辑器（VS Code、Word）的行为一致。
>
> Redo 操作仍然是 O(1)；清空 Redo Stack 在最坏情况下是 O(k)（k 为 redo 栈中的操作数），但这个代价分摊到每次 type 操作上是可接受的。
